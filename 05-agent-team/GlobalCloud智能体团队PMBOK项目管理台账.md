# GlobalCloud 智能体团队PMBOK项目管理台账

日期：2026-06-07
状态：PMBOK 项目管理台账 v1
用途：作为 GlobalCloud 智能体团队执行 `GlobalCloud 绿色供应链体系` 时的项目管理总台账。

## 1. 台账使用说明

本台账用于把 PMBOK 管理要求固定成可跟踪字段。
更新责任人：小即。
专项输入责任人：各专业成员。
审计复核责任人：评衡、证验。

## 2. PMBOK 管理域台账

| 管理域 | 当前控制对象 | 核心字段 | 当前主责 |
|---|---|---|---|
| 整合管理 | 总体阶段、交付包、变更、阶段结论 | 当前阶段、总体状态、阶段目标、变更项 | 小即 |
| 范围管理 | 交付边界、变更项 | 范围基线、范围偏差、未批准新增项 | 小即 / 各成员 |
| 进度管理 | 阶段计划、周计划、里程碑 | 计划开始、计划结束、偏差、延期原因 | 小即 |
| 成本管理 | token、时间、返工成本 | token 消耗、返工轮次、重复改写次数 | 小即 / 评衡 |
| 质量管理 | 质量门禁、审计结果 | 当前得分、未闭合缺陷、退回项 | 评衡 / 证验 |
| 资源管理 | 智能体负载与分工 | 成员分工、当前任务量、交付包占用 | 小即 |
| 沟通管理 | 汇报、周报、控制塔 | 日报、周报、升级汇报、更新时间 | 小即 |
| 风险管理 | 风险与阻塞 | 风险编号、等级、缓解动作、关闭状态 | 小即 / 各成员 |
| 外部依赖管理 | 外部系统、连接器、人工确认 | 依赖项、状态、责任人、阻塞影响 | 小即 / 接稳 |
| 相关方管理 | 用户与各成员的确认关系 | 需确认事项、确认状态、关闭日期 | 小即 |
| 模型治理 | 模型选择、升级、返工与成本控制 | 默认主模型、默认副模型、禁止模型、升级记录、返工原因 | 小即 / 评衡 |

## 4. PMBOK 偏差指标

当前团队固定追踪以下偏差指标：

1. 范围偏差项数
2. 进度偏差项数
3. 成本偏差项数
4. 质量未闭合项数
5. 风险未闭合项数
6. 外部依赖阻塞项数
7. 相关方待确认项数
8. 模型升级项数
9. 模型返工项数
10. 高质量模型使用占比

## 5. 台账使用规则

1. 所有重大新增事项必须先进范围台账，再进入任务派发。
2. 所有延期必须进入进度台账。
3. 所有返工必须进入成本台账。
4. 所有审计退回必须进入质量台账。
5. 所有重大阻塞必须进入风险或外部依赖台账。
6. 所有人工确认事项必须进入相关方确认台账。
7. 所有模型升级和模型误用返工必须进入模型治理台账。
