## 3. 体系架构与治理主线

这条主线回答“整个体系怎么设计、怎么分层、怎么治理”。

当前已明确：

1. 体系名称为 **GlobalCloud 绿色供应链体系**
2. 主业务架构采用三层：
   - 治理与监控层
   - 运营与协同层
   - 生产与执行层
3. 横向底座采用 **AI 与数据层**
4. `WAES` 是治理、证据、状态、发布验证、连接器治理、知识准入、AI 授权中枢
5. `GPC-Native` 是运营与协同层主线
6. `GFIS` 是生产与执行层主线和工厂事实主账
7. `Edge` 是现场接入层，不是业务主账
8. `PVAOS` 是生态入口
9. 不把 `WAES` 写成事务审批中心
10. 不把 AI 建议写成业务事实

这一主线已经落成对象目录、事件合同、验收矩阵、四流分析、连接器合同、SOP 模板库、AI 服务模型、数据治理模型等一整套设计基线。

## 9. 工作区/项目仓库/环境治理主线

这条主线回答“哪些工作在总工作区做，哪些工作必须进真实仓库做”。

当前已明确：

1. 总工作区负责：
   - 总控
   - 台账
   - 计划
   - 周报
   - 汇总
   - 状态控制
2. 真实项目仓库负责：
   - 代码
   - 配置
   - 运行态验证
   - 连接器验证
   - Git 与 dirty state 证据
3. 进入真实项目仓库前必须通过只读预检
4. 既有 dirty state 不得被覆盖

这条线解释了为什么“可以实施”和“已经可以联调”不是一回事。
