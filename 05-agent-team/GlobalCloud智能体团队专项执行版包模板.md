# GlobalCloud 智能体团队专项执行版包模板

日期：2026-06-08
状态：专项执行版包模板 v1
用途：供左侧主线程在不改变总体口径的前提下，把过程控制规范压入各专项实施包。
适用范围：宪衡、链同、厂行、数枢、知源、灵策、评衡、证验。

## 1. 使用说明

本模板只用于把“专项首轮实施前验证包”升级为“专项执行版包”。

执行版包必须同时绑定以下 5 类规范：

1. 交付物完成判定规范
2. 项目仓库实施准入规范
3. 设计-实现追踪矩阵规范
4. 测试与验证规范
5. 状态升级与验收放行规范

本模板不得用于：

1. 替代真实仓库预检结论
2. 替代真实验证证据
3. 直接把专项状态升级为 `ready_for_human_acceptance` 或 `complete`

## 3. 各专项套用建议

### 3.1 宪衡

重点补：

1. `EvidenceRecord / StatusAudit / AgentToolGrant`
2. 规则门、状态门、证据门
3. WAES 首轮运行前验证项

### 3.2 链同

重点补：

1. PVAOS 入口对象落点
2. GPC 原型对象与接口落点
3. `GPC` 一期蓝图与验证包

### 3.3 厂行

重点补：

1. GFIS 主链对象与接口
2. Edge 联合验证样本
3. 执行闭环回执要求

### 3.4 数枢

重点补：

1. 数据库权限
2. 读模型只读
3. DLQ / Replay / Trace / Evidence

### 3.5 知源

重点补：

1. Brain remote
2. ingest / 引用回指 / 失效拦截
3. A19 验收包

### 3.6 灵策

重点补：

1. Agent 权限
2. AISuggestion 回执
3. 越权拦截与主账无写入样本

### 3.7 评衡

重点补：

1. 评分复评模型与结果闭环
2. 缺口优先级和治理阻塞归类
3. 阶段门禁差异判定（设计 / 实施前准备 / 联调前）

### 3.8 证验

重点补：

1. 证据分级与归档规则
2. 证据样本与状态升级联动
3. 人工验收前置条件的门禁定义

## 4. 左侧主线程使用建议

左侧主线程应按以下顺序套用：

1. 先用本模板升级 `宪衡专项首轮实施前验证包`
2. 再复制到 `链同 / 厂行 / 数枢 / 知源 / 灵策 / 评衡 / 证验`
3. 由小即统一检查 6 个先导专项 + 4 个待启动显性会话的执行版包是否满足：
   - 实施准入
   - 追踪闭合
   - 完成判定
   - 验证清单
   - 状态放行条件

## 5. 当前结论

本模板属于**侧线程补充的执行方法资产**。

它不推进主线程状态，不改变真实仓库，不改 Linear 主台账，只为左侧主线程提供直接可套用的执行版骨架。
