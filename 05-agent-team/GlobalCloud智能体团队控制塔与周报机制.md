# GlobalCloud 智能体团队控制塔与周报机制

日期：2026-06-07  
状态：控制塔与周报机制 v1  
适用范围：GlobalCloud 智能体团队的日常项目控制、周报汇总、阶段复评和人工验收前状态观察。  
管理口径：同时遵循 Harness Engineering 与 PMBOK。

## 1. 目标

本机制用于让用户通过一个统一控制塔掌握：

1. 总体项目进度
2. 交付包推进状态
3. 各智能体工作状态
4. 当前评分与质量状态
5. 当前风险、阻塞和需人工确认事项
6. PMBOK 范围、进度、成本、质量和风险偏差
7. 模型使用是否符合团队质量与成本控制机制

## 2. 控制塔模块

总体控制塔固定包含以下模块：

1. 总体阶段面板
2. 交付包进度面板
3. 智能体团队状态面板
4. 质量与评分面板
5. 风险与阻塞面板
6. 证据与状态面板
7. 需人工确认事项面板
8. PMBOK 管理视图面板

## 3. 总览字段

控制塔总览必须显示：

| 字段 | 含义 |
|---|---|
| 当前阶段 | 当前所处阶段 |
| 当前总体状态 | Harness 总体状态 |
| 设计基线分 | 设计基线评分 |
| 实施准备分 | 实施准备评分 |
| 运行准备分 | 运行准备评分 |
| 交付包完成率 | 已完成/应完成的交付包比率 |
| 当前高风险项数 | 高风险项总数 |
| 当前阻塞项数 | 当前阻塞项总数 |
| 需人工确认事项数 | 待人工确认项数 |
| 是否达到 `ready_for_human_acceptance` | 当前是否具备人工验收准备 |
| 范围偏差项数 | 范围扩散或未受控变更数 |
| 进度偏差项数 | 当前延期或里程碑偏差数 |
| 成本偏差项数 | token、时间、返工成本偏差数 |
| 质量未闭合项数 | 质量缺陷未闭合项数 |
| 风险未闭合项数 | 风险未闭合项数 |
| 当前主要使用模型 | 当前阶段主导使用的模型 |
| 模型升级项数 | 本周期触发模型升级的次数 |

## 4. 下钻字段

### 4.1 按交付包下钻

| 字段 | 含义 |
|---|---|
| 交付包名称 | 当前交付包 |
| 牵头成员 | 当前主责智能体 |
| 当前状态 | Harness 状态 |
| 本周完成工作 | 当前周已完成的内容 |
| 当前任务 | 正在推进的任务 |
| 下一步计划 | 下一阶段动作 |
| 主要困难 | 关键问题 |
| 阻塞项 | 当前阻塞 |
| 风险等级 | 高/中/低 |
| PMBOK 偏差 | 当前包的范围/进度/成本/质量/风险偏差 |
| 当前主要使用模型 | 当前包主要使用的模型 |
| 是否触发模型升级 | 是否发生从低成本模型到高质量模型的升级 |
| 升级原因 | 模型升级的直接原因 |

### 4.2 按成员下钻

| 字段 | 含义 |
|---|---|
| 成员名称 | 小即/宪衡/链同等 |
| 负责交付包 | 当前成员牵头范围 |
| 当前状态 | 当前状态 |
| 本周完成工作 | 已完成工作 |
| 当前任务 | 进行中事项 |
| 下周计划 | 下一步工作 |
| 主要困难 | 困难说明 |
| 当前阻塞项 | 阻塞说明 |
| 需协调事项 | 需要小即或用户协调事项 |
| 当前 PMBOK 管理问题 | 当前管理偏差 |
| 当前主要使用模型 | 当前成员主要使用的模型 |
| 模型使用异常 | 是否存在误用高成本或误用低成本模型 |

## 5. 更新频率

### 5.1 每日更新字段

1. 当前状态
2. 当前任务
3. 当前阻塞项
4. 主要困难
5. 需协调事项

### 5.2 每周更新字段

1. 本周完成工作
2. 下周计划
3. 交付包完成率
4. 当前评分
5. 高风险项
6. PMBOK 偏差项
7. 模型升级项
8. 模型返工项

### 5.3 阶段更新字段

1. 当前阶段
2. 阶段完成结论
3. 是否进入 `ready_for_human_acceptance`
4. 是否建议阶段升级
5. 阶段质量未闭合项
6. 模型治理异常项

## 6. 周报机制

## 6.1 小即周报

小即每周必须对用户提交：

1. 总体项目进展
2. 交付包推进状态
3. 各成员状态摘要
4. 当前风险与阻塞
5. 需人工确认事项
6. 当前阶段升级建议

## 6.2 成员周报

每个成员每周必须向小即提交：

1. 本周完成工作
2. 当前任务
3. 下周计划
4. 主要困难
5. 阻塞项
6. PMBOK 管理问题

## 8. 使用规则

1. 所有成员不得直接对用户提交独立项目结论，统一经小即汇总。
2. 控制塔字段必须可追溯到实际任务、文档、证据或阶段结论。
3. 未经审计通过的阶段结论，不得写入“已完成”。
4. `ready_for_human_acceptance` 只能由小即基于审计与证据结论向用户建议。
5. 模型使用必须遵守《GlobalCloud智能体团队模型使用治理与成本控制机制》。
