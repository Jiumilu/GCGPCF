# GlobalCloud PKC-KDS 个人知识与贡献入口清单

日期：2026-06-17
状态：v0.1 受控入口清单
适用范围：PKC、KDS、GPCF、WAES、Brain、GFIS / GCFIS、GPC、PVAOS、XiaoC / 小即、XGD、XiaoG、MMC，以及葛化物流和湖北磷材绿色供应链试点。

## 1. 定位

本文是 `GPCF-KDS-DKS-016` 的主交付，用于明确 PKC 与 KDS 在分布式知识事实体系中的边界。

总原则是：KDS 管全局事实，PKC 管个人参与。KDS 负责全局知识事实、11 个底座资源池、增强治理账本、可信来源、权限密级、WAES 门禁和 Loop 证据；PKC 负责个人知识、个人任务、个人贡献、个人积分候选、悬赏参与、个人 AI 服务入口和合作单位成员空间。

本文只定义 PKC 如何接入 KDS 11 个底座资源池和增强治理账本，不表示真实账号已经创建、真实人员已经入驻、真实积分已经确认、真实收益已经分配或真实 AI 额度已经开通。

## 2. 一句话边界

| 事项 | KDS | PKC |
|---|---|---|
| 全局事实 | 保存候选事实、确认事实、证据索引、底座资源池挂接 | 不保存为全局事实主账 |
| 个人参与 | 记录全局引用和确认状态 | 承载个人知识、任务、提交、积分候选和 AI 服务入口 |
| 贡献积分 | 保存贡献账本、积分池、确认状态和争议记录 | 展示个人候选积分、确认积分、冻结积分、扣减和争议 |
| 悬赏交易 | 保存悬赏池、验收、冻结、结算和争议 | 参与悬赏、提交材料、查看自己范围内的结果 |
| 权限密级 | 维护 DSR、组织、角色、合作单位可见范围 | 只按授权展示个人和被邀请任务范围 |
| AI 服务 | 记录额度池、使用计量、来源和挂接 | 提供个人 AI 服务入口、额度余额和使用记录 |
| 决策裁决 | 保留委员会、WAES 和人工确认记录 | 不裁决积分、收益、争议和重大违规 |

## 3. PKC 可承载的入口

| 入口 | 说明 | 进入 KDS 的方式 | 必要约束 |
|---|---|---|---|
| 个人知识 | 个人整理的项目、订单、行业、原料、政策、SOP 知识 | 作为 KnowledgeCandidate 挂接数据池 / 场景池等 | 必须保留来源、提交人和密级 |
| 个人任务 | 分配给个人的补证、验收、整理、答疑、悬赏任务 | 作为 PersonalTask 关联对应底座资源池 | 不等于任务已完成 |
| 个人贡献 | 个人提交的知识、证据、纠错、渠道、产值线索、治理建议 | 进入 ContributionCandidate | 确认前只作候选积分 |
| 个人积分 | 知识积分、产值积分、渠道积分、悬赏积分、治理积分等 | 进入积分池 / 贡献账本增强对象 | 必须区分候选、确认、冻结、争议、撤销 |
| 个人悬赏 | 个人参与系统或合作单位发起的知识缺口悬赏 | 进入悬赏池和提交记录 | 验收前不得结算 |
| 个人 AI 服务 | 个人使用平台额度、自购额度或共享额度 | 进入额度池使用记录 | 自购额度先自用，不进入统一收益池 |
| 合作单位成员空间 | 合作单位下人员、角色、任务、贡献和可见范围 | 挂接人才池、数据池、场景池 | 合作单位默认只能看自身明细 |

## 4. 个人对象流

```text
PersonalAction
  -> PKCIntakeRecord
  -> KDSCandidateRegistration
  -> BaseResourcePoolLink
  -> EnhancedLedgerLink
  -> WAESBoundaryAndEvidenceGate
  -> HumanOrCommitteeDecision
  -> PersonalLedgerView
  -> KDSBacklinkAndLoopEvidence
```

### 4.1 PersonalAction

个人动作可以来自：

1. 上传或整理知识材料。
2. 提交会议、电话、邮件、第三方文档、网络搜索或业务资料摘要。
3. 参与知识缺口、证据缺口或悬赏任务。
4. 领取 GFIS / GPC / PVAOS / Brain / KDS 相关个人任务。
5. 使用个人 AI 服务进行摘要、问答、SOP 候选、缺口候选或写回候选生成。
6. 提交纠错、争议、申诉、补证和撤回请求。

### 4.2 PKCIntakeRecord

PKC 入口记录必须包含：

| 字段 | 要求 |
|---|---|
| pkcRecordId | PKC 入口记录统一编号 |
| actorType | person / unit_member / project_member / committee_member |
| actorId | 人员或成员标识 |
| organizationId | 所属合作单位或项目组 |
| actionType | knowledge_submit / task_submit / bounty_submit / ai_use / correction / dispute |
| sourceRefs | 来源索引 |
| baseResourcePools | 至少一个 KDS 底座资源池 |
| enhancedLedgerLinks | 关联积分池、悬赏池、额度池、争议池、贡献账本等 |
| classificationLevel | DSR-L0 / DSR-L1 / DSR-L2 / DSR-L3 |
| trustLevel | T0 / T1 / T2 / T3 |
| confirmationStatus | draft / candidate / submitted / reviewed / confirmed / rejected / withdrawn |
| waesGateRefs | 规则、证据、权限或例外门禁引用 |
| loopEvidenceRefs | GPCF Loop 记录引用 |

## 5. KDS 11 池挂接规则

| PKC 对象 | 必挂底座资源池 | 可挂增强账本 | 说明 |
|---|---|---|---|
| 个人知识 | 数据池、人才池、场景池 | 贡献账本、积分池 | 个人知识只作候选，确认后进入全局知识事实索引 |
| 个人任务 | 人才池、数据池、场景池 | SOP 账本、贡献账本 | 任务状态不等于业务事实完成 |
| 订单线索贡献 | 订单池、数据池、场景池 | 潜在产值池、贡献账本、积分池 | 未到账前只能作为潜在产值 |
| 产能 / 工厂贡献 | 产能池、装备池、数据池、场景池 | SOP 账本、贡献账本 | 可服务葛化和湖北磷材拓厂复制 |
| 原料 / 行业贡献 | 原料池、政策池、数据池 | 知识缺口悬赏池、贡献账本 | 适用于湖北磷材原料 / 行业知识库 |
| 物流 / POD 贡献 | 运力池、订单池、数据池 | 证据对象、贡献账本、争议池 | POD 仍需业务系统确认 |
| 金融凭证贡献 | 资金池、订单池、数据池 | 收益池、权限账本、争议池 | DSR-L3 默认元数据或封存索引 |
| AI 额度使用 | 资金池、数据池、人才池、场景池 | 额度池 | 自购额度先自用，可计量但不进入统一收益池 |
| 悬赏提交 | 数据池、人才池、场景池，按问题挂接相关资源池 | 悬赏池、积分池、争议池 | 验收前不得结算 |
| 争议 / 申诉 | 人才池、数据池、资金池、场景池 | 争议池、贡献账本 | 委员会多数决，重大事项备案 |

## 6. 积分状态机

```text
candidate
  -> submitted
  -> waes_checked
  -> reviewed
  -> confirmed
  -> allocated
```

冻结和异常分支：

```text
candidate
  -> returned_for_evidence
  -> frozen
  -> disputed
  -> rejected
  -> withdrawn
  -> retroactive_deduction
```

规则：

1. 候选积分不等于确认积分。
2. 有实际收入的贡献可列入产值；无实际收入只能列入知识或潜在产值。
3. 正式收入按到账确认；统计和财务过程口径可按开票记录。
4. 一般违规可酌情溯源扣除；重大违规可按事实比例或全部扣除。
5. 争议、扣减、冻结和撤销必须进入争议池并保留 DecisionRecord。

## 7. 悬赏机制

知识缺口积分可以由系统、合作单位、项目组、人员或委员会发起。PKC 负责个人参与入口，KDS 负责全局挂接和状态记录。

| 阶段 | PKC 动作 | KDS / WAES / 委员会动作 |
|---|---|---|
| 发起 | 查看悬赏、申请参与、提交问题补充 | 登记悬赏池对象，挂接底座资源池 |
| 提交 | 上传材料、来源、摘要、证据索引 | KDS 生成候选记录，WAES 做规则和证据门禁 |
| 验收 | 查看个人提交状态和反馈 | 项目组或委员会验收 |
| 结算 | 查看候选积分、冻结积分、确认积分 | 进入积分池 / 贡献账本，必要时进入收益池 |
| 争议 | 发起申诉、补证或撤回 | 进入争议池，委员会多数决并备案 |

悬赏可以允许多人提交、部分采纳、协商分配、项目组内部分配和跨单位邀请，但必须保留来源、责任主体、验收记录和权限边界。

## 8. 个人 AI 服务入口

PKC 可以为个人提供 AI 服务入口，但 AI 只生成候选输出。

| AI 场景 | 输出 | 进入流程 |
|---|---|---|
| 个人知识整理 | 摘要、标签、来源清单、知识候选 | KDS 候选登记 |
| 任务辅助 | 任务拆解、补证清单、SOP 候选 | WAES / 人工确认 |
| 悬赏提交辅助 | 资料结构化、缺口解释、证据建议 | 悬赏池候选提交 |
| 订单线索辅助 | 需求摘要、潜在产值候选、字段建议 | 订单池 / 潜在产值池候选 |
| 纠错辅助 | 错误定位、来源对照、修正建议 | 争议池或纠错候选 |

额度规则：

1. 平台统一提供 AI 服务。
2. 合作单位和个人未来可自购额度、贡献额度、共享额度。
3. 自购额度先自用，可计量但不进入统一收益池。
4. 共享额度和平台额度需记录来源、使用人、用途、成本、余额、挂接池和 Loop 证据。

## 9. 合作单位成员空间

合作单位成员空间用于让葛化物流、湖北磷材及后续合作单位的成员参与知识、任务、积分、悬赏和 AI 服务。

| 空间对象 | 说明 | 边界 |
|---|---|---|
| MemberProfile | 成员身份、角色、所属单位、权限范围 | 不自动获得他方资料 |
| MemberTask | 成员任务、补证、审核、整理、提交 | 完成状态需证据 |
| MemberKnowledge | 成员知识草稿、来源、摘要、标签 | 确认前只是候选 |
| MemberContribution | 成员贡献候选、确认、冻结、扣减 | 需进入贡献账本 |
| MemberBounty | 成员参与悬赏、提交、验收、争议 | 验收前不得结算 |
| MemberAICredit | 成员 AI 额度来源、使用、余额 | 自购额度先自用 |

合作单位可以看到自己的成员、任务、贡献和收益候选明细；他方明细需要邀请、授权、悬赏参与或委员会规则允许。

## 10. 试点映射

### 10.1 葛化物流

PKC 在葛化第一阶段服务：

1. 葛化成员提交建设资料、工厂运营资料、订单资料、质量 / 发货 / POD / 金融凭证元数据。
2. 现代精工 OEM 过渡资料的责任拆分、补证、纠错和任务分配。
3. 辽宁远航链路的知识缺口、证据缺口、潜在产值和悬赏参与。
4. GFIS 知识问答助手、GFIS 使用助手、GFIS 文档验收助手的个人提问、反馈和缺口提交。
5. 预运营期订单母版中的个人任务、补证记录和候选贡献。

PKC 不确认葛化订单、生产、质量、POD、到账、开票、收益或正式积分。

### 10.2 湖北磷材

PKC 在湖北磷材第一阶段服务：

1. 拓厂项目知识提交、补证、任务分配和成员协同。
2. 原料 / 行业 / 订单知识库的个人知识贡献。
3. 新工厂复制模板的个人经验、SOP 候选和纠错建议。
4. 与葛化母版复用相关的知识缺口悬赏和个人贡献候选。

湖北磷材第一阶段不做 GFIS 深度运行，PKC 主要服务拓厂知识、原料行业知识、订单线索和新工厂复制模板的个人参与。

## 11. 不得越界清单

1. PKC 不替代 KDS 全局事实底座。
2. PKC 不替代 GFIS / GCFIS、GPC、PVAOS 等来源系统主账。
3. PKC 不替代 WAES 规则、证据、权限和例外门禁。
4. PKC 不裁决积分、收益、争议和重大违规。
5. PKC 不把个人提交、AI 输出、悬赏提交或任务完成写成业务事实确认。
6. PKC 不绕过合作单位可见范围、DSR 密级、用户治理权和委员会机制。
7. PKC 不把自购 AI 额度纳入统一收益池。

## 12. Definition of Done

本轮满足以下条件时，DKS-016 才算完成：

1. KDS 与 PKC 的全局事实和个人参与边界已明确。
2. 个人知识、个人任务、个人积分、个人悬赏、个人 AI 服务和合作单位成员空间入口已定义。
3. PKC 对象已挂接到 KDS 11 个底座资源池和增强治理账本。
4. 候选积分、确认积分、冻结、争议、撤销和追溯扣减状态已定义。
5. 葛化物流和湖北磷材试点中的 PKC 角色已明确。
6. 文档进入 README、文档控制台账、KDS 开发空间同步台账和 `.kds` 本地镜像。
7. 污染检查、KDS TOKEN 检查和 Loop 文档门禁通过。
