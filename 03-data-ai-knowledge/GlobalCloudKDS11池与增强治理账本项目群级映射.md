# GlobalCloud KDS 11 池与增强治理账本项目群级映射

日期：2026-06-20
轮次：`GPCF-KDS-DKS-059`
状态：`controlled_pool_ledger_mapping`

## 1. 状态边界

本文把 KDS 11 个底座资源池与积分池、收益池、额度池、悬赏池、争议池、潜在产值池、贡献账本和 SOP 账本进行项目群级映射。本文不新增底座池，不确认真实积分、收益、额度、悬赏、争议、到账、开票、质量、POD 或业务完成。

## 2. 底座 11 池

| 底座池 | 定义 | 典型事实 | P0 允许状态 |
|---|---|---|---|
| 订单池 | 订单、需求、报价、客户确认、交付闭环 | 预运营期订单、订单线索、正式订单缺口 | candidate / repair_required |
| 运力池 | 物流、承运、发货、签收、POD | 发货计划、承运记录、POD 索引 | metadata_only / candidate |
| 产能池 | 工厂、产线、OEM、排产、产能调度 | 葛化目标工厂、现代精工 OEM、复制工厂 | candidate |
| 资金池 | 开票、到账、财务凭证、收益池 | 开票统计、到账候选、收益候选 | metadata_only / candidate |
| 政策池 | 政策、标准、合规、权威来源 | 政府政策、标准网站、行业规则 | limited / safe_after_gate |
| 装备池 | 设备、模具、工艺装备、产线 | 拓厂设备、OEM 设备、产线能力 | candidate |
| 数据池 | 文档、证据、台账、报告、RAG、LOOP | KDS 文档、证据、评测、缺口 | controlled / candidate |
| 能源池 | 能耗、碳、绿色能源、节能资料 | 能耗指标、碳资料、节能方案 | candidate |
| 原料池 | 原料、供应商、价格、批次、质量 | 磷材原料、PP/改性料、供应线索 | candidate / sensitive_metadata_only |
| 人才池 | 责任人、审核人、专家、委员会、岗位能力 | 记录人、项目负责人、委员会成员 | candidate / confirmed_by_human |
| 场景池 | 项目、工厂复制、区域运营、应用场景 | 葛化母版、湖北磷材、新工厂复制 | template_candidate / controlled |

## 3. 增强治理账本

增强账本不是底座池，必须挂接到底座池。

| 账本 | 作用 | 必挂池 | P0 状态 |
|---|---|---|---|
| 积分池 | 知识、产值、渠道、治理、纠错、悬赏积分 | 人才池、数据池、场景池 | candidate |
| 收益池 | 知识收益、系统收益、业务收益 | 资金池、订单池、场景池 | candidate |
| 额度池 | 平台、自购、贡献、共享、奖励 AI 额度 | 资金池、数据池、人才池 | registered / candidate |
| 悬赏池 | 知识缺口、证据缺口、SOP 缺口悬赏 | 数据池、人才池、场景池 | bounty_candidate |
| 争议池 | 积分、收益、悬赏、权限、证据争议 | 人才池、数据池、资金池 | open / frozen |
| 潜在产值池 | 未到账业务机会、订单线索、拓厂机会 | 订单池、资金池、场景池 | potential_value_candidate |
| 贡献账本 | 资料、补证、SOP、渠道、治理贡献 | 人才池、数据池、场景池 | candidate |
| SOP 账本 | 全局 SOP、分段 SOP、岗位 SOP、复制模板 | 场景池、数据池 | suggestion_only / controlled_template |

| 质量/POD/金融门禁 | 运力池、资金池、数据池 | 争议池、贡献账本 | metadata-only 不等于放行 |

| RAG 准入 | 数据池、政策池、场景池 | 贡献账本 | repair/blocked 不得强引用 |

## 6. 收益与积分挂接口径

| 场景 | 可计入 | 不可计入 |
|---|---|---|
| 无收入资料贡献 | 知识积分候选、贡献账本 | 产值积分、收益池 |
| 订单线索但未到账 | 潜在产值池、渠道积分候选 | 正式收入、收益分配 |
| 已开票未到账 | 财务统计过程口径 | 正式收益池 |
| 已到账收入 | 收益池候选、产值积分候选 | 未经确认的分配 |
| 合作单位自购 AI 额度 | 额度池计量、自用记录 | 统一收益池 |
| 平台或贡献额度 | 额度池、贡献账本 | 未备案收益分配 |

## 7. 门禁

| 门禁 | 通过条件 | 不通过处理 |
|---|---|---|
| 底座池挂接 | 至少一个底座池，且符合对象类型 | returned_for_evidence |
| 来源门禁 | 有 sourceRefs 和责任主体 | repair_required |
| 证据门禁 | 有 evidenceRefs 或缺口说明 | repair_required |
| 密级门禁 | DSR-L2/L3 资料有脱敏或 sealed 规则 | blocked |
| WAES 门禁 | 有 gate result 或待 gate 说明 | manual_confirmation_required |
| 收益门禁 | 有到账证据才允许正式收益候选 | blocked |
| 委员会门禁 | 重大事项有 DecisionRecord 路径 | committee_review_required |
| RAG 门禁 | trustLevel、evidence、classification 均符合 | limited / repair_required / blocked |

## 8. LOOP 证据要求

每条增强账本对象必须能回指：

1. 触发来源。
2. KDS 底座池。
3. WAES 门禁。
4. 人工或委员会路径。
5. 状态和不能升级原因。
6. 对应 LOOP round。

## 9. 当前禁止事项

1. 不得把增强账本写成底座资源池。
2. 不得把候选积分写成确认积分。
3. 不得把开票写成到账。
4. 不得把潜在产值写成正式产值。
5. 不得把自购 AI 额度写入统一收益池。
6. 不得把悬赏候选写成已发布或已结算。
7. 不得把委员会模板写成委员会已裁决。
