# GlobalCloud 绿色供应链合作单位接入与组织空间初始化清单

日期：2026-06-17
状态：v0.1 受控初始化清单
适用范围：葛化、湖北磷材、未来类似葛化物流有限公司的工厂建设单位、区域绿色供应链运营单位、上下游服务单位。

## 1. 定位

本文是 `GPCF-KDS-DKS-013` 的专项产物，用于把合作单位复制接入绿色供应链分布式知识系统的初始化动作拆成可审计、可回放、可追责的清单。

本文只定义接入准备、组织空间、账号、资料包、密级、AI 额度、积分账户、悬赏账户、KDS / WAES / GFIS / Hermes 接入边界和 Loop 门禁，不创建真实账号、不配置生产权限、不开放真实资料、不表示任何合作单位已经完成接入。

## 2. 初始化原则

1. 葛化作为第一阶段标准母版，湖北磷材作为第二条并行线。
2. 每家合作单位默认拥有独立组织空间、独立资料归属、独立 AI 额度账户、独立积分账户和独立悬赏参与记录。
3. KDS 11 池统一承载知识、事实、证据、SOP、积分、收益、潜在产值、悬赏、AI 额度、治理和权限对象。
4. 合作单位可见本单位资料、本单位积分和额度、被邀请事项、公开规则和参与的悬赏事项。
5. AI 服务由平台统一提供；合作单位自购 AI 额度先自用，不进入统一收益池。
6. WAES 只确认规则、证据、权限和边界；GFIS / GPC / PVAOS 继续作为对应业务事实主责系统。
7. 任何 DSR-L2 / DSR-L3 资料进入 AI 使用、导出、共享、降密或收益分配前，必须有人工确认或委员会备案。
8. 接入过程必须纳入 Loop 工程治理，不能把文档完成写成业务接入完成。

## 4. 组织空间初始化

|---|---|---|
| 飞书组织 / 群组 | 单位项目群、资料协作群、会议记录入口、授权联系人 | 平台 + 合作单位管理员 |
| 小即接入空间 | AI 服务入口、助手调用入口、使用日志摘要、额度显示 | 平台 AI 服务负责人 |

| 悬赏管理员 | 建议 | own_unit + invited | 发起或参与知识缺口悬赏 |

## 6. 密级初始化

| 资料类型 | 默认密级 | 默认处理 |
|---|---|---|
| 公开政策 / 标准 / 行业资料 | DSR-L0 | 可进入 AI 引用，需保存来源和检索日期 |
| 一般培训、流程、SOP 草案 | DSR-L1 | 项目组内可用 |
| 订单、客户需求、项目会议、合作单位资料 | DSR-L2 | 只用脱敏摘要进入 AI |
| 合同、价格、金融凭证、到账、收益分配、重大争议 | DSR-L3 | AI 只读元数据或不使用原文 |
| 葛化预运营期订单 | DSR-L2 | 目标工厂与 OEM 承接方责任分开记录 |
| 质量 / 发货 / POD 证据 | DSR-L2 / DSR-L3 | 按字段拆分，敏感字段元数据化 |
| 湖北磷材拓厂评估 | DSR-L1 / DSR-L2 | 前期以知识和潜在产值为主 |

初始化时宁可高密级起步，后续按 WAES 规则或委员会备案降密。

## 8. AI 额度初始化

| 额度类型 | 是否进入收益池 | 默认使用 |
|---|---:|---|
| 平台提供额度 | 否 | 试点、内测、治理、首批问答 |
| 合作单位自购额度 | 否 | 本单位先自用 |
| 合作单位贡献额度 | 视备案 | 可进入共享或悬赏奖励 |
| 共享额度 | 视规则 | 支持项目组或被邀请事项 |
| 悬赏奖励额度 | 否 | 按悬赏结算结果发放 |

每次 AI 使用至少记录：使用人、单位、场景、额度来源、受益对象、是否含敏感资料、是否可计贡献、是否进入悬赏或知识缺口。

## 9. 积分与收益初始化

| 项 | 初始化动作 |
|---|---|
| 候选积分账户 | 为合作单位建立知识、产值、渠道、治理、证据、复用、悬赏类型的候选积分账户 |
| 确认积分账户 | 只接收规则确认或委员会确认后的积分 |
| 潜在产值池 | 记录未到账但可追踪的业务机会 |
| 收益池 | 只接收到账后的正式收益；开票用于统计和财务过程口径 |
| 自购额度计量 | 可计量但不进入收益池 |
| 悬赏资源冻结 | 发布悬赏前冻结积分、现金、AI 额度或服务权益 |
| 扣分追溯 | 一般问题酌情扣除，重大违规按事实比例或全部扣除 |

初始阶段知识和产值权重更高，但具体权重和兑换系数必须版本化，可随阶段、项目和市场调整。

## 10. 接入流程

```text
合作单位确认
  -> 组织空间登记
  -> 角色与授权登记
  -> 密级矩阵初始化
  -> 首批资料包登记
  -> AI 额度账户初始化
  -> 积分 / 悬赏账户初始化
  -> KDS 11 池挂接
  -> WAES 规则记录
  -> 内测或资料验收
  -> 接入验收记录
```

每一步都必须形成 `source_refs`、责任人、状态、下一步动作和是否需要人工确认。

## 13. 葛化优先动作

1. 以葛化作为订单运行母版和 GFIS AI 助手三件套试点。
2. 先建立 `KPK-GH-BLD`、`KPK-GH-OPS`、`KPK-GH-ORD`、`KPK-GH-LY`、`KPK-GH-OEM`、`KPK-GH-QPOD`、`KPK-GH-FIN` 七类资料包。
3. 按 DSR-L2 / DSR-L3 拆分辽宁远航、现代精工 OEM、质量、发货、POD、金融凭证门禁资料。
4. 先运行知识问答、使用助手、文档验收助手的内测样本，不开放真实业务确认。
5. 从内测输出中提取 AI 建议、SOP 建议、知识缺口和悬赏候选。

## 14. 湖北磷材优先动作

1. 不做 GFIS 深度运行，先建立拓厂项目知识库、原料 / 行业 / 订单知识库和新工厂复制模板。
2. 以葛化母版作为参考，但必须保留湖北磷材独立单位、独立资料包和独立贡献记录。
3. 对拓厂资料先计知识贡献和潜在产值贡献，不确认正式收入。
4. 对订单线索建立潜在产值记录，到账后才进入正式收益池。
5. 对原料和行业资料建立可信级别，权威政策 / 标准源可进入更高可信级别。
