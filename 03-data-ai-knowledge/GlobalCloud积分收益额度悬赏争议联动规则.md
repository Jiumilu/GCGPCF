# GlobalCloud 积分收益额度悬赏争议联动规则

状态：`GPCF-KDS-DKS-020` 受控规则  
日期：2026-06-17  
适用范围：GlobalCloud 项目群、KDS 11 个底座资源池、积分池、收益池、额度池、悬赏池、争议池、潜在产值池、贡献账本、SOP 账本、葛化物流试点、湖北磷材试点、后续工厂建设与区域绿色供应链运营单位。

## 1. 定位

本文是 `GPCF-KDS-DKS-020` 的正式交付，用于把积分池、收益池、额度池、悬赏池、争议池、潜在产值池、贡献账本和 SOP 账本统一成一套可审计、可回放、可追责、可复制的联动规则。

本文不新建底座资源池。订单池、运力池、产能池、资金池、政策池、装备池、数据池、能源池、原料池、人才池、场景池才是 KDS 11 个底座资源池。积分、收益、额度、悬赏、争议、潜在产值、贡献和 SOP 均为增强治理账本对象，必须至少挂接一个底座资源池。

本文不确认真实积分、真实收益、真实额度、真实悬赏结算、真实争议裁决或真实业务完成。AI 只能生成候选事实、候选 SOP、候选积分和候选写回建议，所有候选必须进入 KDS / WAES / 人工或委员会确认流程。

## 2. 总原则

1. 所有增强账本对象必须挂接 KDS 11 个底座资源池，并保留来源、责任主体、确认状态、权限密级、WAES 门禁记录和 LOOP evidence。
2. 候选积分不等于确认积分；候选收益不等于到账收益；候选悬赏不等于已发布悬赏；候选争议不等于已裁决争议。
3. 无实际收入时只能计知识贡献、渠道贡献、治理贡献、悬赏贡献或潜在产值贡献；有实际收入且到账后，才可进入正式收入和收益池分配口径。
4. 开票用于统计和财务过程口径，到账用于正式收入和正式收益分配口径。
5. 自购 AI 额度先自用，可计量但不进入统一收益池；贡献额度、共享额度、平台额度和奖励额度必须分别登记。
6. 一般违规可酌情溯源扣除；重大违规可按事实比例或全部扣除，并可冻结相关积分、收益、额度、悬赏或可见范围。
7. 用户保留体系治理权、急停权和规则治理权；委员会负责积分、收益、悬赏、争议和重大违规裁决。

## 3. 增强账本与底座池挂接

| 增强账本 | 必挂底座资源池 | 可选底座资源池 | 典型对象 |
|---|---|---|---|
| 积分池 | 人才池、数据池、场景池 | 订单池、产能池、资金池、政策池、装备池、原料池、运力池 | 知识积分、产值积分、渠道积分、悬赏积分、治理积分、纠错积分 |
| 收益池 | 资金池、订单池、场景池 | 产能池、原料池、运力池、装备池、人才池 | 到账收入、收益分配、冻结份额、转正记录 |
| 额度池 | 资金池、数据池、人才池、场景池 | 订单池、政策池 | 平台额度、自购额度、共享额度、贡献额度、奖励额度、使用计量 |
| 悬赏池 | 数据池、人才池、场景池 | 订单池、原料池、产能池、政策池、装备池、运力池 | 知识缺口悬赏、证据缺口悬赏、SOP 缺口悬赏、纠错悬赏 |
| 争议池 | 人才池、数据池、资金池、场景池 | 相关事实所在底座池 | 积分争议、收益争议、悬赏争议、证据争议、权限争议、违规争议 |
| 潜在产值池 | 订单池、资金池、场景池 | 产能池、原料池、运力池、人才池 | 订单线索、拓厂机会、未到账业务机会、辽宁远航链路、湖北磷材订单线索 |
| 贡献账本 | 人才池、数据池、场景池 | 与贡献事实相关的全部底座池 | 知识、证据、SOP、渠道、产值、治理、补证、纠错贡献 |
| SOP 账本 | 场景池、数据池 | 订单池、产能池、运力池、装备池、原料池、资金池 | 全局 SOP、分段 SOP、岗位 SOP、预运营期订单 SOP、拓厂复制 SOP |

## 4. 统一对象字段

所有增强账本对象至少包含：

| 字段 | 说明 |
|---|---|
| ledgerObjectId | 统一编号，如 `LEDGER-{类型}-{单位}-{日期}-{序号}` |
| ledgerType | point / revenue / credit / bounty / dispute / potential_value / contribution / sop |
| sourceProject | 来源项目、合作单位或系统 |
| sourceObjectRefs | 资料、订单、会议、邮件、网络来源、AI 输出、证据包、任务或 LOOP 记录 |
| baseResourcePools | 至少一个 KDS 11 底座资源池 |
| enhancedLedgerLinks | 相关增强账本交叉引用 |
| responsibleSubject | 责任主体，可为个人、单位、项目组或委员会 |
| affectedParties | 受影响主体 |
| classificationLevel | DSR-L0 / DSR-L1 / DSR-L2 / DSR-L3 |
| trustLevel | T0 / T1 / T2 / T3 |
| confirmationStatus | candidate / submitted / waes_checked / human_review_pending / committee_review_pending / confirmed / allocated / frozen / disputed / withdrawn / rejected / retroactive_deduction |
| waesGateRefs | 规则、证据、权限、边界或例外记录 |
| decisionRecordRefs | 委员会或人工确认记录 |
| loopEvidenceRefs | GPCF LOOP 证据 |
| reversible | 是否允许撤回、冲销、追溯扣减或重新确认 |

## 5. 统一状态机

```text
candidate
  -> submitted
  -> waes_checked
  -> human_review_pending
  -> committee_review_pending
  -> confirmed
  -> allocated
```

异常和保护分支：

```text
candidate
  -> returned_for_evidence
  -> frozen
  -> disputed
  -> withdrawn
  -> rejected
  -> retroactive_deduction
```

| 状态 | 触发条件 | 处理 |
|---|---|---|
| returned_for_evidence | 来源、证据、责任主体或字段不足 | 退回补证，可进入悬赏池 |
| frozen | 权限、收益、积分、事实或争议未清 | 暂停分配、写回和外部可见 |
| disputed | 任何主体提出异议 | 进入争议池，必要时委员会多数决 |
| withdrawn | 提交方撤回或系统发现来源不成立 | 保留撤回记录，不再确认 |
| rejected | WAES、人工或委员会确认不符合规则 | 保留原因，不进入确认积分或收益 |
| retroactive_deduction | 一般或重大违规成立 | 按规则扣减、冲销或冻结 |

## 6. 积分规则

| 积分类型 | 触发对象 | 默认挂接 | 初始阶段建议 |
|---|---|---|---|
| 知识积分 | 文档、会议、行业、政策、原料、SOP 候选 | 数据池、人才池、场景池 | 权重较高 |
| 产值积分 | 已到账收入对应贡献 | 订单池、资金池、场景池 | 只在到账后正式确认 |
| 渠道积分 | 客户、供应商、合作单位、区域线索 | 人才池、订单池、场景池 | 可先作为候选 |
| 潜在产值积分 | 未到账但可追踪业务机会 | 订单池、资金池、场景池 | 可累计，待到账转正 |
| 悬赏积分 | 知识缺口、证据缺口、SOP 缺口完成 | 数据池、人才池、场景池 | 验收后确认 |
| 治理积分 | 规则完善、验收、审计、门禁、复核 | 数据池、人才池、场景池 | 需避免自我确认 |
| 纠错积分 | 发现事实错误、权限错误、污染或越权风险 | 数据池、争议池相关挂接池 | 可奖励，也可触发扣减 |

权重和兑换系数必须版本化。不同阶段、项目、市场情况或贡献类型可以使用不同兑换系数，由 MMC 维护参数基线，重大调整需备案。

## 7. 收益规则

| 收益类型 | 进入条件 | 口径 |
|---|---|---|
| 知识收益 | 知识产品、知识服务或悬赏服务产生实际收入 | 到账后进入正式收益池 |
| 系统收益 | AI 服务、平台服务、系统服务产生实际收入 | 到账后进入正式收益池 |
| 业务收益 | 订单、工厂、供应链业务产生实际收入 | 到账后进入正式收益池 |
| 统计收益 | 已开票但未到账 | 只作统计和财务过程口径 |
| 潜在收益 | 线索、报价、意向、拓厂机会 | 进入潜在产值池，不进入正式收益池 |

收益分配参考贡献积分、产值积分、渠道积分、治理贡献、悬赏贡献和争议处理结果，但最终分配必须有到账证据、WAES 规则记录和人工或委员会确认。

## 8. 额度规则

| 额度类型 | 是否进入统一收益池 | 处理 |
|---|---:|---|
| 平台提供额度 | 否 | 用于试点、内测、治理和首批问答，记录成本和使用对象 |
| 合作单位自购额度 | 否 | 先自用，可计量，不进入统一收益池 |
| 个人自购额度 | 否 | 个人先自用，可计量，不进入统一收益池 |
| 贡献额度 | 视备案 | 可作为共享资源或悬赏奖励，需记录来源和用途 |
| 共享额度 | 视规则 | 支持项目组、被邀请事项或悬赏任务 |
| 悬赏奖励额度 | 否 | 作为奖励发放，需记录结算依据 |

每次 AI 使用至少记录：使用人、单位、额度来源、使用场景、受益对象、是否含敏感资料、是否可计贡献、是否关联悬赏、是否进入知识缺口。

## 9. 悬赏规则

| 阶段 | 输入 | 输出 | 门禁 |
|---|---|---|---|
| 发起 | 知识缺口、证据缺口、SOP 缺口、纠错需求 | 悬赏候选 | KDS 候选登记 |
| 发布 | 发起方、范围、奖励、验收标准、冻结资源 | 已发布悬赏 | WAES 规则记录 |
| 提交 | 材料、来源、摘要、证据索引 | 悬赏提交候选 | 密级和权限检查 |
| 验收 | 提交质量、完整性、来源、可复用性 | 通过、部分通过、退回、驳回 | 人工或委员会 |
| 结算 | 积分、额度、服务权益或收益参考 | 结算记录 | 争议期后分配 |
| 争议 | 质量、先后、比例、权限或重复提交争议 | 争议池记录 | 委员会多数决 |

悬赏允许多人提交、部分采纳、协商分配、项目组内部分配和跨单位邀请，但必须保留来源、责任主体、验收记录、权限边界和 LOOP evidence。

## 10. 争议与违规规则

| 类型 | 典型场景 | 临时措施 | 最终处理 |
|---|---|---|---|
| 积分争议 | 多人提交同一知识、贡献比例不清 | 冻结候选积分 | 比例拆分、确认、驳回或退回 |
| 收益争议 | 收益分配比例、到账归属、贡献权重争议 | 冻结争议份额 | 分配、暂缓、撤销或复核 |
| 悬赏争议 | 提交质量、先后顺序、重复提交争议 | 暂缓结算 | 结算、拆分、退回或驳回 |
| 证据争议 | 真实性、完整性、权属不清 | 降级为候选事实 | 补证、确认或撤销 |
| 权限争议 | 密级、可见范围、脱敏程度争议 | 按更高密级保护 | 可见范围决定 |
| 一般违规 | 引用错误、资料不完整、轻微误判 | 酌情冻结或扣减 | 补证后可恢复 |
| 重大违规 | 伪造证据、泄密、越权确认、虚构收入 | 立即冻结相关对象 | 按事实比例或全部扣除，并备案 |

涉及重大违规、收益分配、潜在产值转正式产值、密级争议或跨单位权益争议时，必须进入委员会机制。

## 11. 潜在产值转正式产值

```text
订单线索 / 拓厂机会 / 渠道贡献
  -> 潜在产值候选
  -> WAES 规则检查
  -> 人工或委员会确认
  -> 持续跟踪开票和到账
  -> 到账后转正式收入
  -> 收益池分配参考
```

| 阶段 | 可计入 | 不可计入 |
|---|---|---|
| 线索阶段 | 知识贡献、渠道贡献、潜在产值候选 | 正式收入、正式产值积分 |
| 报价/开票阶段 | 统计和财务过程口径、潜在产值追踪 | 正式收益分配 |
| 到账阶段 | 正式收入、正式产值积分、收益池分配参考 | 未经确认的贡献比例 |
| 争议阶段 | 冻结份额、争议记录 | 外部分配和最终确认 |

## 12. 葛化与湖北磷材试点映射

| 试点 | 进入积分池 | 进入潜在产值池 | 进入收益池 | 进入悬赏池 | 进入争议池 |
|---|---|---|---|---|---|
| 葛化建设资料 | 建设知识、SOP、补证贡献 | 无或项目机会候选 | 不直接进入 | 建设资料缺口 | 责任、密级、权属争议 |
| 葛化预运营期订单 | 订单知识、补证、SOP 贡献 | 辽宁远航等未到账机会 | 到账后进入 | 客户确认、POD、质量缺口 | OEM 责任、质量、POD、收益争议 |
| 葛化金融凭证 | 元数据整理、脱敏、门禁贡献 | 开票或待到账机会 | 到账后进入 | 收益缺口 | 金额、到账、可见范围争议 |
| 湖北磷材拓厂 | 拓厂知识、模板、评估贡献 | 拓厂机会和区域机会 | 到账后进入 | 拓厂资料缺口 | 权重、贡献比例、复用争议 |
| 湖北磷材原料/行业/订单 | 原料、行业、订单线索贡献 | 订单线索和供应链机会 | 到账后进入 | 原料、行业、订单资料缺口 | 价格、供应商、客户线索争议 |

## 15. 阶段收口建议

`GPCF-KDS-DKS-014` 至 `GPCF-KDS-DKS-020` 已形成分布式 KDS 知识事实体系的受控规则骨架。下一阶段建议进入：

1. 葛化 DKS-018 的首批资料包样本表和 AI 输出评分记录模板。
2. 湖北磷材 DKS-019 的拓厂评估样本表和原料/行业/订单知识源目录。
3. DKS-020 的规则参数表，由 MMC 管理权重、兑换系数、额度类型和冻结/扣减参数。
4. 继续保持不创建真实账号、不配置生产权限、不写生产数据、不调用真实外部 API、不升级 accepted/integrated 的边界。
