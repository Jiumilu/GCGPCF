# 宪衡专项审批阻断与证据写入链样本

日期：2026-06-08
专项：宪衡
范围：WAES 治理与控制塔域
当前状态：`in_progress`

## 1. 目标

抽取 WAES 中“审批阻断 -> 审批请求 -> 证据写入 -> 工具状态更新”的链式样本结构，确认高风险 Agent 工具不会直接执行，而是先进入治理链。

## 2. 主要实现入口

已确认文件：

1. `src/app/agent-runtime/draftApprovalRuntime.ts`

## 3. 链式样本结构

### 3.1 高风险工具先进入 `waiting_approval`

已确认：

1. `execute()` 中先调用 `options.ledger.recordToolCall()`
2. 初始写入状态为：`waiting_approval`

已确认样本语义：

1. 高风险工具不会直接进入 `running`
2. 工具调用先形成 `toolCall`
3. 工具调用先发布 `agent.tool.requested`

### 3.2 先生成 draft evidence

已确认：

1. `draftEvidence = await recordEvidence(...)`

已确认 evidence 样本语义：

1. `type: tool_call`
2. `title: ${contract.name} draft receipt`
3. `source: toolId`
4. `relatedIds` 中可包含：
   - `toolId`
   - `toolCallId`
   - `workflowId`
   - `packageId`
   - `packageVersion`
5. `inputHash = hashAgentEvidenceValue(input)`
6. `outputHash = hashAgentEvidenceValue(draft)`

当前判断：

- 高风险工具即使只是在 draft 阶段，也已经先沉淀第一笔 evidence。

### 3.3 再生成 approval request

已确认：

1. `approvalRequest = await createApprovalRequest(...)`
2. `createApprovalRequest()` 内部调用 `options.ledger.recordApprovalRequest(...)`

已确认审批请求样本语义：

1. `status: pending`
2. `riskLevel`
3. `targetAction`
4. `impactScope`
5. `evidenceIds: [draftEvidence.id]`
6. `expiresAt = addHours(..., policy.slaHours)`

当前判断：

- approval request 不是孤立对象，而是显式引用了 draft evidence。

### 3.4 再生成 approval evidence

已确认：

1. `approvalEvidence = await recordEvidence(...)`

已确认 evidence 样本语义：

1. `type: approval`
2. `title: Approval requested for ${contract.name}`
3. `source: agent-draft-approval-runtime`
4. `relatedIds` 中包含：
   - `toolId`
   - `toolCallId`
   - `approvalRequestId`
5. `outputHash` 明确写入：
   - `{ status: 'pending', approvalRequestId }`

当前判断：

- WAES 不是只记录审批请求对象，还会把“审批请求已产生”单独写成 approval evidence。

### 3.5 最后更新 tool call 仍保持 `waiting_approval`

已确认：

1. `options.ledger.updateToolCall(...)`
2. 状态仍然是：`waiting_approval`
3. `outputSummary` 明确写出：
   - approval request 已创建
   - no external dispatch was attempted

当前判断：

- WAES 在审批前明确保持工具调用停留在 `waiting_approval`
- 没有借机把状态拔高成执行中

## 4. 审批决定后的样本结构

### 4.1 批准/拒绝决定

已确认：

1. `decideApproval(input)`
2. 内部再次调用 `recordApprovalRequest(...)`
3. 更新字段：
   - `status: approved | rejected`
   - `decidedBy`
   - `decidedAt`

### 4.2 决定后 evidence

已确认：

1. `recordEvidence(...)`
2. `type: approval`
3. `title: Approval approved / rejected`
4. `summary` 会记录：
   - approvalRequestId
   - actorId
   - rationale

当前判断：

- 审批决定本身也被证据化。

## 5. 审批不可执行时的阻断样本

### 5.1 已批准且有效时

已确认：

1. `checkApprovalForExecution()` 返回：
   - `status: executable`

### 5.2 审批过期时

已确认：

1. 会再次写 `recordApprovalRequest(...)`
2. 生成 evidence：
   - `title: Expired approval blocked execution`
   - `requiresNewApprovalRequest: true`
3. 返回：
   - `status: blocked`

### 5.3 审批非可执行时

已确认：

1. 会生成 evidence：
   - `title: Approval blocked execution`
   - `failureReason: approval_not_executable`
2. 返回：
   - `status: blocked`
   - `reason: approval_not_executable`

当前判断：

- WAES 已实现“审批过期阻断”和“审批不可执行阻断”的双路径样本。

## 6. 当前链式结论

当前已经确认的治理链是：

1. `toolCall(waiting_approval)`
2. `draftEvidence(tool_call)`
3. `approvalRequest(pending)`
4. `approvalEvidence(approval)`
5. `toolCall update(waiting_approval, no external dispatch)`
6. `approval decided / expired / blocked`
7. `decision evidence / blocked evidence`

这说明：

- WAES 已实现完整的“高风险工具不得直接执行”的治理链样本。
- 当前缺的不是链路结构，而是真实运行中产生的一条具体记录样本。

## 7. 当前仍缺的硬样本

1. 真实 `toolCall` 记录样本
2. 真实 `approvalRequest` 记录样本
3. 真实 `draftEvidence` 样本
4. 真实 `approvalEvidence` 样本
5. 真实 `Approval blocked execution` 样本
