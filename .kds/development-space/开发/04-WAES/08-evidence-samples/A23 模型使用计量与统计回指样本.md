# A23 模型使用计量与统计回指样本

日期：2026-06-08
状态：首轮运行前样本 v1
对应验收：A23 模型使用计量与统计回指验证

## 1. 验证目标

验证 `ModelUsageRecord` 与 `ModelMeterSnapshot` 在当前实现中的等价入口、字段映射和回指链是否存在，并明确当前仍缺的 live 返回样本。

## 2. 当前结论

当前 `A23` 已确认：

1. `hermes-webui` 已存在 usage 统计总入口。
2. 单次 session 级 token / cost 字段已存在持久化与同步入口。
3. 汇总统计可以回指到模型维度，再回指到 session 维度。

当前 `A23` 仍未完成：

1. 尚未取得一次真实 `insights` HTTP 返回样本。
2. 尚未取得一次真实 `state.db.sessions` 查询返回样本。
3. 尚未形成“统计结果 -> 单条 session -> 运行上下文”的实测截图或导出证据。

## 3. `ModelMeterSnapshot` 等价入口

主入口：

- `/Users/lujunxiang/Projects/hermes-webui/api/routes.py:3690`

函数：

- `_handle_insights(handler, parsed)`

该入口当前输出：

- `period_days`
- `total_sessions`
- `total_messages`
- `total_input_tokens`
- `total_output_tokens`
- `total_tokens`
- `total_cost`
- `models`
- `daily_tokens`
- `activity_by_day`
- `activity_by_hour`

判断：

这已经构成 `ModelMeterSnapshot` 的实现等价入口，尤其是：

1. 总量统计
2. 按模型分组统计
3. 按日统计
4. 活跃度统计

## 4. `ModelUsageRecord` 等价来源

### 4.1 WebUI session index 来源

在 `_handle_insights()` 中，先从：

- `SESSION_DIR / "_index.json"`

聚合 session 条目。

读取字段包括：

- `input_tokens`
- `output_tokens`
- `estimated_cost`
- `message_count`
- `model`
- `created_at`
- `updated_at`

### 4.2 CLI / Agent session 数据库来源

在同一函数中，还会读取：

- `state.db`
- 表：`sessions`

SQL 读取字段：

- `id`
- `model`
- `message_count`
- `input_tokens`
- `output_tokens`
- `estimated_cost_usd`
- `started_at`
- `ended_at`

判断：

这构成 `ModelUsageRecord` 的 session 级等价来源。

## 5. 使用记录同步入口

同步入口：

- `/Users/lujunxiang/Projects/hermes-webui/api/routes.py:6544`

关键调用：

- `sync_session_usage(...)`

当前同步字段：

- `session_id`
- `input_tokens`
- `output_tokens`
- `estimated_cost`
- `model`

这说明 usage 不是只在展示层计算，而是有单独的会话级同步入口，适合作为 `ModelUsageRecord` 的等价写入链依据。

## 6. 会话对象中的 usage 字段

## 9. 回指链

当前可确认的回指链为：

```text
总计量快照
-> models[]
-> 单模型 bucket
-> session index / state.db.sessions 单条记录
-> session_id / started_at / ended_at / title / profile
```

这条链在结构上已经成立，但还缺 live 样本来证明：

1. 某个 `models[]` 统计值
2. 能稳定回指到某一批 session 记录
3. 这些 session 记录能被再次定位到运行上下文

## 10. 当前缺口

当前仍缺以下硬样本：

1. 一次 `insights` API 真实返回样本
2. 一次 `_index.json` session usage 样本
3. 一次 `state.db.sessions` 查询样本
4. 一次“模型统计 -> session 记录”回指证据
5. 若要进入更高阶段，还需要补：
   - `traceId`
   - `evidenceRefs`
   - 项目维度或 Agent 维度 usage 归集字段

## 11. 当前结论

`A23` 已经从“纯设计要求”推进到“实现入口和字段映射已确认”的状态。
当前可判定为：

- 有 `ModelMeterSnapshot` 等价入口
- 有 `ModelUsageRecord` 等价来源
- 有结构性回指链

但还不能判定：

- 计量统计已实测通过
