# 宪衡专项连接器阻断与状态链样本

日期：2026-06-08
专项：宪衡
范围：WAES 治理与控制塔域
当前状态：`in_progress`

## 1. 目标

抽取 WAES 中连接器 dispatch 的只读阻断与状态链样本，确认当前不会绕过治理门直接写业务主账。

## 2. 主要实现入口

已确认文件：

1. `src/app/bus/commandBus.ts`
2. `src/app/bus/eventBus.ts`

## 3. CommandBus 样本

### 3.1 命令定义

已确认：

1. `request-connector-dispatch`

对应结果类型：

1. `ConnectorDispatchBlockedResult`

已确认返回字段：

1. `status: blocked`
2. `requestId`
3. `reason`
4. `readonly: true`
5. `request`

### 3.2 只读阻断语义

已确认常量：

1. `READONLY_DISPATCH_REASON`

已确认消息内容：

1. `Connector dispatch is blocked by the Global Cloud CommandBus mock readonly policy; business ledgers are not mutated.`

当前判断：

- WAES 当前明确声明 connector dispatch 在 mock readonly policy 下被阻断。
- 并明确声明不会变更业务主账。

## 4. 事件链样本

### 4.1 发起请求事件

已确认事件：

1. `connector.dispatch.requested`

已确认字段：

1. `requestId`
2. `request`
3. `requestedAt`
4. `sourceCommandId: request-connector-dispatch`

### 4.2 回执事件

已确认事件：

1. `connector.dispatch.receipt`

已确认字段：

1. `status: blocked | accepted | running`
2. `reason`
3. `readonly: true`
4. `receiptAt`
5. `receiptSource: mock_readonly_policy | remote_api`
6. `failureReason`

当前判断：

- 虽然当前命令实现返回的是 `blocked`，但 event bus 合同已经为 `accepted/running` 保留状态位。
- 这意味着当前阶段是只读阻断样本，不代表未来运行态只能 blocked。

### 4.3 运行反馈事件

已确认事件：

1. `runtime.feedback.received`

已确认字段：

1. `workflowId`
2. `relatedScenarioId`
3. `title`
4. `status`
5. `sourceTopic`
6. `detail`
7. `receivedAt`
8. `failureReason`

已确认当前 mock 语义：

1. `status: degraded`
2. `sourceTopic: pvaos.runtime.dispatch.receipt.v1`
3. detail 明确说明：
   - 目标平台返回只读调度回执
   - 当前不会写入业务主账
   - 需保留失败回退说明

## 5. 当前连接器状态链

当前已确认的状态链是：

1. `request-connector-dispatch`
2. `connector.dispatch.requested`
3. `connector.dispatch.receipt(status=blocked, readonly=true)`
4. `runtime.feedback.received(status=degraded)`

当前判断：

- WAES 已实现连接器调度请求、回执和运行反馈三段链。
- 当前链路明确是“只读阻断 + 风险回退说明”，没有伪装成成功 dispatch。

## 6. EventBus 合同样本

已确认 `eventBus.ts` 中定义：

1. `ConnectorDispatchRequestedEvent`
2. `ConnectorDispatchReceiptEvent`
3. `RuntimeFeedbackReceivedEvent`

其中：

1. `ConnectorDispatchReceiptEvent.status` 允许：
   - `blocked`
   - `accepted`
   - `running`
2. `RuntimeFeedbackReceivedEvent.status` 走 `GlobalCloudRuntimeFeedbackStatus`

当前判断：

- WAES 在合同层已经保留了未来真实 remote API 回执的扩展位。
- 当前阻断样本只是当前实现边界，不应被误写成未来唯一状态。

## 7. 当前仍缺的硬样本

1. 真实 `accepted` 回执样本
2. 真实 `running` 回执样本
3. 真实 `remote_api` receiptSource 样本
4. 真实 connector runtime feedback 样本

## 8. 当前结论

宪衡专项连接器链已经确认：

1. 当前 dispatch 默认受 readonly policy 阻断
2. 当前不会写业务主账
3. 当前会留下 request / receipt / runtime feedback 三段证据链

下一步不必再证明“是否存在连接器阻断语义”，而应继续寻找真实 remote/accepted/running 的运行前样本。
