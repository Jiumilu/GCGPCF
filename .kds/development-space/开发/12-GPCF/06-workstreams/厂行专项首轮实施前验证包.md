# 厂行专项首轮实施前验证包

日期：2026-06-10
专项：厂行
范围：GFIS 工厂执行 + Edge 联合域
当前状态：`in_progress`
基线引用：[GlobalCloud绿色供应链体系总架构.md](/Users/lujunxiang/Projects/GlobalCloud%20GPCF/GlobalCloud绿色供应链体系总架构.md)

## 0. 口径修正（与 [总架构] 一致）

1. 本验证包以 GFIS 作为工厂执行系统 / 工厂事实主账的唯一主责系统。
2. 仓库中 `gcfis_custom` / `ERPNext` 资产归为 GFIS 历史实现资产 / 适配器资产 / legacy reference，不作为本验证包的主线定义来源。
3. Edge 优先服务 GFIS；Edge 不得绕过 GFIS 先进入 GPC 公共服务数据池。
4. 厂行侧连接器落到 5 个根连接器编号下：`CON-GPC-GFIS-001` / `CON-GPC-WAES-001` / `CON-GFIS-WAES-001` / `CON-EDGE-GFIS-001` / `CON-EDGE-WAES-001`。
5. `CON-GFIS-WAES-001` 不废弃，作为 GFIS ↔ WAES 工厂治理 / 异常 / 审计 / 证据直连连接器保留。

## 1. 当前验证目标

1. 抽取 GFIS 工单、质量、库存、批次、发货、维护相关主链对象的运行前样本入口。
2. 明确 LES / EAM 当前实现面，并把当前实现面与 GFIS 主线定义解耦。
3. 明确 Edge 到 GFIS 的事实转换、补传和回执样本要求。
4. 验证 5 根连接器中厂行侧连接器（`CON-GPC-GFIS-001` / `CON-GFIS-WAES-001` / `CON-EDGE-GFIS-001` / `CON-EDGE-WAES-001`）的字段与状态。
5. 验证 GFIS 主线定义与 `gcfis_custom` / `ERPNext` legacy reference 之间的边界。

## 2. 当前边界

1. 本包覆盖 GFIS 工厂执行事实主链与 Edge 联合域。
2. Edge 当前没有独立仓库证据，因此只按联合口径推进。
3. 本包只定义运行前样本要求，不做工厂执行闭环完成判断。
4. 本包不得把 `gcfis_custom` / `ERPNext` 资产的运行结果当作 GFIS 主线的运行结果。

## 9. 当前完成判定与状态上限

仅当以下条件全部满足时，才可判定厂行专项首轮实施前验证包准备完成：

1. GFIS 主链对象与 Edge 联合样本要求都形成"实现落点 + 验证入口"。
2. 5 根连接器中厂行侧连接器形成"字段完整 + 状态明确 + 真实事件回执"。
3. `gcfis_custom` / `ERPNext` legacy reference 与 GFIS 主线边界分离形成可追溯记录。

当前状态上限：

1. 最高只能保持 `partial`
