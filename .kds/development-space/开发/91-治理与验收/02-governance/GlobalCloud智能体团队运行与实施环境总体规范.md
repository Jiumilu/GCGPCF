# GlobalCloud智能体团队运行与实施环境总体规范

日期：2026-06-08
状态：环境总体规范 v1
适用范围：GlobalCloud 绿色供应链体系的设计、实施、验证、试运行、验收与运行治理全过程。
治理口径：同时遵循 Harness Engineering、PMBOK、当前三层主架构和 AI 与数据层边界。

## 1. 目的

本规范用于解决以下问题：

1. 明确整个实施与交付过程到底在什么环境中工作。
2. 明确不同环境允许做什么、禁止做什么。
3. 明确不同环境能产出什么证据。
4. 明确不同环境的状态上限，防止把低层证据误写成高层完成。

## 2. 环境分层模型

整个体系的运行与实施环境统一分为 6 层：

1. 设计控制环境
2. 项目仓库环境
3. 本地开发运行环境
4. 集成验证环境
5. 试运行环境
6. 生产运行环境

## 3. 六层环境定义

### 3.1 设计控制环境

典型位置：

- `/Users/lujunxiang/Documents/GlobalCloud智慧工厂`

主要用途：

1. 体系架构设计
2. 对象目录、事件合同、验收矩阵设计
3. 团队控制塔、PMBOK 台账、实施计划、周报
4. 多专项汇总、评估、审计、状态控制

允许动作：

1. 文档设计
2. 方案收口
3. 台账汇总
4. 目标、计划、控制与风险分析

禁止动作：

1. 把文档存在写成系统完成
2. 把设计证据写成运行证据
3. 把本环境写成真实运行环境

可产出证据：

1. 设计证据
2. 规划证据
3. 控制证据
4. 状态判断建议

状态上限：

- `in_progress`
- 设计完备度可 100/100
- 不得进入联调完成、试运行完成、生产完成表述

### 3.2 项目仓库环境

典型位置：

- `WAES`
- `PVAOS`
- `GFIS`
- `GPC`
- `Brain`
- `XiaoC`
- `Hermes`
- `XGD`

主要用途：

1. 仓库级只读预检
2. 分支、remote、dirty state、对象落点、接口落点核查
3. 实际代码、配置、脚本、目录、运行入口确认

允许动作：

1. Git preflight
2. 只读检查
3. 必要时在获准后做最小实现改动

禁止动作：

1. 未经明确批准擅自改写代码
2. 覆盖既有 dirty state
3. 用仓库级证据直接宣称联调完成

可产出证据：

1. Git 证据
2. 实现落点证据
3. 仓库级预检结论

状态上限：

- `partial`
- 可进入“实施前验证准备”
- 不得直接进入 `ready_for_human_acceptance`

### 3.3 本地开发运行环境

主要用途：

1. 单仓库本地启动
2. 开发态配置验证
3. 本地 UI / API / 脚本运行检查

允许动作：

1. 本地启动
2. 本地 smoke
3. 单仓库功能验证

禁止动作：

1. 把单仓库本地运行写成跨系统联调完成

可产出证据：

1. 本地运行证据
2. 本地 smoke 证据

状态上限：

- 本地运行通过
- 不得直接等同系统联调通过

### 3.4 集成验证环境

主要用途：

1. 连接器联调
2. 跨系统 API / Event / 证据链验证
3. 数据库权限、读模型、DLQ、Replay、知识发布链验证

允许动作：

1. 联调
2. 集成 smoke
3. 运行前样本验证

禁止动作：

1. 无完整证据就宣称试运行通过

可产出证据：

1. 联调证据
2. 接口回执
3. 事件链与证据链样本

状态上限：

- 可进入 `ready_for_human_acceptance` 前置判断

### 3.5 试运行环境

主要用途：

1. 最小闭环试运行
2. 日报、异常、复盘、治理跟踪
3. 人工验收前稳定性验证

可产出证据：

1. 试运行证据
2. 闭环证据
3. 人工验收前证据

状态上限：

- 可进入 `ready_for_human_acceptance`
- 需人工确认后才能 `accepted`

### 3.6 生产运行环境

主要用途：

1. 正式运行
2. 正式监控、审计、治理、发布与复盘

可产出证据：

1. 生产运行证据
2. 稳定运行指标
3. 正式审计与经营侧证据

状态上限：

- `accepted`
- 在满足正式闭环条件后，才可能进入 `complete`

## 4. 环境门规则

统一环境门如下：

1. 没离开设计控制环境，不能产出运行证据。
2. 没进入项目仓库环境，不能产出仓库级预检证据。
3. 没进入本地开发运行环境，不能产出本地运行证据。
4. 没进入集成验证环境，不能产出联调证据。
5. 没进入试运行环境，不能产出试运行通过证据。
6. 没进入生产运行环境，不能产出生产完成证据。
