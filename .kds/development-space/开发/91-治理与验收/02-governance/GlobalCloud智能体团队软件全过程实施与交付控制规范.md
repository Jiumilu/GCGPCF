# GlobalCloud智能体团队软件全过程实施与交付控制规范

日期：2026-06-08
状态：全过程实施与交付控制规范 v1

## 1. 目的

本规范用于把整个软件工作全过程做成统一控制链，覆盖：

1. 目标
2. 设计
3. 环境
4. 仓库预检
5. 实施
6. 验证
7. 试运行
8. 验收
9. 发布
10. 运行治理
11. 复盘与知识回流

## 2. 全过程阶段模型

### 阶段 1：目标与范围确定

输入：

1. 用户目标
2. 当前基线
3. 项目范围

输出：

1. 目标模式要求
2. 范围边界
3. 交付物清单

### 阶段 2：体系设计与控制基线

输入：

1. 架构目标
2. 专项范围

输出：

1. 架构设计
2. 对象目录
3. 事件合同
4. 验收矩阵
5. 控制塔与 PMBOK 台账

### 阶段 3：环境与工具规范

输出：

1. 环境分层
2. 工作区与仓库规范
3. Codex 工具与技能治理规范

### 阶段 4：真实项目仓库预检

输出：

1. Git 证据
2. 实现落点证据
3. 正式只读预检结论

### 阶段 5：实施前验证准备

输出：

1. 运行前样本要求
2. 验证包
3. 阻塞清单
4. 入口判断

### 阶段 6：联调与集成验证

输出：

1. 连接器联调证据
2. API / Event / Evidence / Trace 证据
3. 读模型、DLQ、Replay 证据

### 阶段 7：试运行

输出：

1. 最小业务闭环
2. 日报、异常、复盘
3. 人工验收前证据

### 阶段 8：验收与发布

输出：

1. 验收结论
2. 发布结论
3. 上线约束

### 阶段 9：运行治理与复盘

输出：

1. 稳态监控
2. 证据归档
3. 经验回流到知识体系

## 3. 每阶段必须具备的控制项

每个阶段都必须明确：

1. 输入
2. 输出
3. 证据
4. 质量门
5. 状态门
6. 人工确认点

## 4. PMBOK 映射

全过程统一映射到：

1. 整合管理：阶段间收口与切换
2. 范围管理：交付边界不漂移
3. 进度管理：里程碑推进
4. 成本管理：token、时间、返工控制
5. 质量管理：质量门与审计
6. 资源管理：多智能体分工
7. 沟通管理：周报、控制塔、专项回报
8. 风险管理：阻塞清单、风险台账
9. 外部依赖管理：连接器、外部系统、环境
10. 相关方管理：用户确认与人工验收

## 5. Harness 映射

全过程统一映射到：

1. 状态明确
2. 证据明确
3. 验收明确
4. 人工确认明确
5. 不越级升级状态

## 6. 当前阶段定位

当前体系所处位置为：

1. 阶段 1 已完成
2. 阶段 2 已完成
3. 阶段 3 本轮补齐
4. 阶段 4 已完成 6 个专项正式只读预检
5. 阶段 5 刚开始
6. 阶段 6-9 尚未进入

## 7. 当前正式结论

当前可以正式表述为：

**体系已具备进入首轮实施前验证准备的全过程控制基础。**

当前不能表述为：

1. 已联调完成
2. 已试运行完成
3. 已发布完成
4. 已生产运行完成
