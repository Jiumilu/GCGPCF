# GlobalCloud 智能体团队模型使用治理与成本控制机制

日期：2026-06-08
状态：模型治理机制 v1
适用范围：GlobalCloud 智能体团队在 Codex 中执行 `GlobalCloud 绿色供应链体系` 相关设计、治理、实施、同步和控制工作的模型选择、升级、审计与成本控制。

## 1. 目标

本机制用于把模型使用规则从“建议”提升为“执行约束”，确保团队在质量、成本和状态纪律之间保持一致。

本机制解决以下问题：

1. 哪些任务必须使用高质量模型。
2. 哪些任务默认使用成本优化模型。
3. 哪些任务禁止使用低成本模型。
4. 模型升级和降级由谁决定。
5. 如何把模型选择纳入控制塔、交付包和 PMBOK 台账。

## 2. 基本原则

团队采用三层模型分工：

1. 主质量模型：`GPT-5.2-Codex`
2. 日常成本模型：`GPT-5 mini`
3. 极轻量辅助模型：`GPT-5 nano`

执行原则：

1. 影响架构、边界、状态、治理、主线代码和验收判断的任务，优先使用 `GPT-5.2-Codex`。
2. 结构明确、低风险、重复性强的任务，默认使用 `GPT-5 mini`。
3. `GPT-5 nano` 仅用于极轻量格式处理和机械整理。
4. 不允许把低成本模型用于状态升级、验收裁决、跨域边界裁决和高风险主线代码修改。

## 3. 模型使用分层

### 3.1 强制高质量任务

以下任务必须使用 `GPT-5.2-Codex`：

1. 顶层架构设计
2. 跨项目治理判断
3. 多文件代码主线改造
4. 复杂调试和定因
5. 状态升级建议
6. `ready_for_human_acceptance` 准入判断
7. 高风险验收结论
8. 关键文档定版

### 3.2 默认成本优化任务

以下任务默认使用 `GPT-5 mini`：

1. 周报整理
2. Linear 同步文案
3. 模板化文档补齐
4. 固定格式台账维护
5. 常规状态汇总
6. 低风险结构化整理

若出现连续返工、边界混乱或质量不稳定，再升级到 `GPT-5.2-Codex`。

### 3.3 极轻量辅助任务

以下任务才允许使用 `GPT-5 nano`：

1. 标题整理
2. 纯格式转换
3. 极轻量摘要
4. 机械型批处理分类

## 4. 禁止规则

以下任务禁止使用 `GPT-5 nano`，并应谨慎限制 `GPT-5 mini`：

1. 是否允许状态升级
2. 是否具备验收条件
3. 是否属于真实完成
4. 是否允许跨系统边界
5. 是否允许 AI 获得某类工具权限
6. 是否允许修改业务主账相关主线实现

以下任务禁止直接由任何模型在无总控确认下自行升级：

1. 从 `in_progress` 升级到 `ready_for_human_acceptance`
2. 从 `ready_for_human_acceptance` 升级到 `accepted`
3. 从 `accepted` 升级到 `complete`

## 5. 升级规则

默认模型升级路径：

1. `GPT-5 mini` -> `GPT-5.2-Codex`

触发条件：

1. 连续两次返工
2. 出现跨域边界冲突
3. 出现架构裁决需求
4. 出现主线代码判断错误
5. 出现状态纪律风险
6. 出现验收结论争议

`GPT-5 nano` 不得直接升级执行治理结论，只能退出并交给 `GPT-5 mini` 或 `GPT-5.2-Codex`。

## 6. 决策权

模型切换权统一归小即总控。

规则如下：

1. 各专项成员可提出模型升级建议。
2. 小即负责最终确认：
   - 是否升级模型
   - 是否保持成本模型
   - 是否因质量或返工退回高质量模型
3. 评衡和证验可对错误模型使用提出审计退回建议。

## 7. 交付包默认模型

## 8. 控制塔字段要求

控制塔和周报必须新增以下字段：

1. 当前主要使用模型
2. 默认模型是否被遵守
3. 是否触发模型升级
4. 升级原因
5. 是否发生返工
6. 模型成本偏差说明

## 9. PMBOK 台账要求

PMBOK 台账必须新增：

1. 模型使用台账
2. 模型升级记录
3. 模型返工记录
4. 高质量模型使用占比
5. 成本模型误用导致的返工项

## 10. 审计规则

评衡与证验必须审查：

1. 是否存在高风险任务误用低成本模型
2. 是否存在机械任务长期误用高质量模型
3. 是否存在因模型选择不当造成返工
4. 是否存在因模型选择不当导致状态漂移

## 11. 最终结论

团队要确保按模型分工执行，不能只靠口头要求，必须把模型规则写进：

1. 交付包责任分解
2. 控制塔与周报机制
3. PMBOK 项目管理台账
4. 审计与返工记录

只有这样，`GPT-5.2-Codex` 才真正承担质量主线，`GPT-5 mini` 才真正承担成本控制主线，而不是停留在建议层。
