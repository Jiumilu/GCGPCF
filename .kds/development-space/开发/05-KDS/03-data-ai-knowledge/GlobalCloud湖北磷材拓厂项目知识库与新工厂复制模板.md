# GlobalCloud 湖北磷材拓厂与新工厂复制试点任务书

状态：`GPCF-KDS-DKS-019` 受控任务书  
日期：2026-06-17  
适用范围：湖北磷材供应链公司、拓厂项目知识库、原料/行业/订单知识库、新工厂复制模板、葛化母版复用、KDS 11 个底座资源池、增强治理账本、WAES 规则门禁、Brain 知识编制、PKC 个人贡献入口、小即问答入口、XGD 候选分析、XiaoG 轻量任务、MMC 参数基线、GPCF LOOP evidence。

## 1. 定位

本文是新队列 `GPCF-KDS-DKS-019` 的正式交付，用于把湖北磷材作为绿色供应链第二条并行线，形成拓厂项目知识库、原料/行业/订单知识库和新工厂复制模板。

湖北磷材代号为 `HBLC`。第一阶段不做 GFIS 深度运行，不创建真实账号，不配置生产权限，不写生产数据，不确认真实订单、原料采购、质量、POD、金融凭证、正式收入或收益分配。AI 只生成候选事实、候选 SOP、候选积分、候选悬赏和候选写回建议，所有候选均需进入 KDS / WAES / 人工确认流程。

本文保留旧队列 `GPCF-KDS-DKS-006` 的有效内容作为历史输入，但当前受控身份按 `GPCF-KDS-DKS-019` 执行。

## 2. 第一阶段目标

1. 建立湖北磷材拓厂项目知识库，用于沉淀目标区域、工厂条件、审批条件、建设计划、装备配置、产能假设和投资边界。
2. 建立原料/行业/订单知识库，用于沉淀原料、供应商、价格线索、行业政策、客户场景、销售订单线索和潜在产值。
3. 建立新工厂复制模板，把葛化母版中的资料包、预运营期订单、OEM 过渡、SOP、知识缺口和悬赏机制转化为 HBLC 可复用结构。
4. 建立 `FactoryExpansionAssessment` 拓厂评估模型，用于排序候选拓厂机会，不作为自动投资、合作、接入或收益分配决策。
5. 将所有知识、事实、证据、SOP、积分、悬赏、潜在产值、权限和 LOOP evidence 挂接到 KDS 11 个底座资源池和增强治理账本。

## 10. 新工厂复制模板

```text
葛化母版输入
  -> HBLC 拓厂项目资料收集
  -> HBLC 原料/行业/订单资料收集
  -> FactoryExpansionAssessment 候选
  -> 新工厂复制模板候选
  -> KDS 11 池与增强账本挂接
  -> WAES 规则记录或阻断
  -> Brain 知识页和复制地图
  -> PKC 任务、贡献和悬赏入口
  -> 用户 / 委员会确认下一阶段
```

复制模板必须保留以下原则：

1. 葛化母版可作为结构输入，但不得把葛化候选事实直接写成湖北磷材事实。
2. 湖北磷材必须保留独立单位、独立资料包、独立贡献记录和独立权限边界。
3. 预运营期订单机制可以复用，但第一阶段只形成候选线索和潜在产值。
4. 新工厂复制模板输出的是候选 SOP、候选任务和候选评估，不代表工厂建设或接入完成。
5. GFIS 第一阶段只做未来接入条件评估，不做运行层深度实施。

## 14. 权限与密级

| 内容 | 默认密级 | 默认处理 |
|---|---|---|
| 公开行业政策 / 标准 | DSR-L0 / DSR-L1 | 可进入 AI 引用，保留来源和检索时间 |
| 一般行业摘要和公开客户场景 | DSR-L1 | 项目组内可用 |
| 原料行情摘要、供应商一般信息 | DSR-L1 / DSR-L2 | 脱敏后可进入知识库 |
| 供应商报价、客户报价、采购订单 | DSR-L2 / DSR-L3 | 默认建立脱敏索引 |
| 拓厂投资边界、资金测算、合同和收益信息 | DSR-L3 | 默认 `metadata_only` 或 `sealed` |
| 跨单位复用葛化母版资料 | DSR-L1 / DSR-L2 | 只复用结构和模板，不复用未确认事实 |

DSR-L3 字段默认只建立受控索引，不进入共享知识库正文，不进入普通 AI 问答。
