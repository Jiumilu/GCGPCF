# GPC 绿色供应链公共服务平台总体方案

日期：2026-06-10
版本：GPC 平台总体方案 v2（口径修正稿）
基线引用：[GlobalCloud绿色供应链体系总架构.md](/Users/lujunxiang/Projects/GlobalCloud%20GPCF/GlobalCloud绿色供应链体系总架构.md)
适用范围：GlobalCloud GPCF 公共服务与治理体系中"绿色供应链公共服务平台"主体的建设、运行、扩展和验收。

## 0. 文档定位

本文是 GPC（Green supply-chain Public-service platform / 绿色供应链公共服务平台）主体的总体方案。GPC 主体在 GlobalCloud GPCF 中的身份、边界、主账、能力、引擎、连接器、事件、对象、SOP、验收均以本文为准；本文与 [总架构] 不一致时，体系内一律以 [总架构] 为基线，本文必须服从 [总架构]。

## 1. 平台身份（与 [总架构] 一致）

1. GPC 平台在 GlobalCloud GPCF 中的身份是 **绿色供应链公共服务平台本体**，是平台主线与公共服务平台本体。
2. GPC 平台 **不是** "公共服务平台"，**不是** "新增中间层"，**不是** 平台订单主账以外的辅助型业务系统，**不是** 工厂执行系统，**不是** 治理审批主账。
3. GPC 平台是"客户、供应商、承运商、监管、园区、运营单位"与"工厂"之间开展绿色供应链协同的公共服务场所，是绿色供应链公共服务在数字空间的本体。
4. GPC 平台内部流程引擎统一命名为 **平台服务流程编排引擎 / 绿色供应链协同流程运行时 / 供应链公共服务流程引擎**，**不得** 再称为"虚拟工厂运行时"。"虚拟工厂"概念只用于 GFIS 内部的工厂执行模型，不属于 GPC 平台本体。

## 2. 平台主责（与 [总架构] §2 一致）

GPC 平台主责且仅主责以下 12 类平台公共服务主账事实：

1. 平台订单（含平台订单主链、订单分发、订单生命周期、订单状态）。
2. 需求池（绿色供应链需求汇聚、需求画像、需求匹配、需求优先级）。
3. 供应商协作（供应商准入、绩效、协作任务、协作证据）。
4. 产能协同（多厂分单、产能承诺、产能快照、产能可见性）。
5. 工厂分配（工厂分配规则、工厂分配结果、分单确认、跨厂协同异常）。
6. ASN（供应商到货通知、ASN 创建、确认、到达、收货状态）。
7. 预约（到货预约、发货预约、月台时段、车辆到达）。
8. 运输（在途节点、运输延迟、运输状态、运输异常）。
9. POD（签收回单、客户签收、签收争议、签收附件）。
10. 异常协同（外部协同异常、跨厂异常、客户/供应商/承运商协同闭环）。
11. 绿色绩效（绿色供应链 KPI 口径、绩效计算、绩效发布）。
12. 合规证据包（绿色合规证据、合规证据确证、合规证据包发布）。

此外，GPC 平台还承担以下 3 类公共服务支撑能力（非主账事实）：

1. 服务开放（平台 API 网关、平台服务目录、平台服务计量）。
2. 生态接入（外部客户、供应商、承运商、监管、园区、银行接入与上下文）。
3. 公共服务（绿色供应链公共服务门户、公共服务订阅、公共服务 SLA）。

GPC 平台 **不主责** 工厂订单、工单、生产报工、质量放行、库存事务、批次、设备维护、发货出库、工厂执行证据；这些事实主账 **必须** 归属 GFIS。

GPC 平台 **不主责** 租户、组织、用户、角色、权限、伙伴档案、项目空间、应用入口、门户菜单、平台运营配置；这些事实主账 **必须** 归属 PVAOS。

GPC 平台 **不主责** 治理规则、证据确证、状态门控、合规校验、风险控制、审计追踪、模型授权、AI 越权拦截、异常升级、验收证据管理；这些事实主账 **必须** 归属 WAES。

GPC 平台 **不承接** 现场信号、报警、缓存、补传作为业务主账事实；这些事实主账归属 GFIS，Edge 不得绕过 GFIS 写 GPC 平台主账。

## 3. 平台与 PVAOS / GFIS / WAES / Edge / Brain / AI 编排层的边界

| 系统 | 在 GPC 平台视角下的角色 | 与 GPC 平台的关系 |
|---|---|---|
| PVAOS | 平台运营、租户、组织、伙伴、用户、权限、门户、项目和应用入口底座 | 上游：通过 `CON-GPC-WAES-001` 治理引用上下文，为 GPC 平台提供租户、组织、用户、角色、伙伴、项目、应用入口 |
| GFIS | 工厂执行系统 / 工厂事实主账 | 下游：通过 `CON-GPC-GFIS-001` 完成订单分发、ASN/预约、运输回传、发货回执、工厂事实回流 |
| WAES | 治理、证据确证、状态门控、合规、风险、审计、AI 越权控制系统 | 旁路：通过 `CON-GPC-WAES-001` 双向引用平台证据、跨厂异常、状态升级请求、阶段验收 |
| Edge | 现场采集与边缘缓冲层 | 间接：Edge 通过 `CON-EDGE-GFIS-001` 服务 GFIS，不直接进入 GPC 平台事实流；可通过 `CON-EDGE-WAES-001` 上送边缘异常 / 安全 / 审计证据（可选） |
| KDS / Brain / LLM Wiki | 企业知识主存 / 知识编制与引擎候选 | 消费：通过 `CON-GPC-WAES-001` 挂接知识准入、版本、发布、引用、AI 可见范围 |
| XiaoC / Hermes / XGD | AI 与 Agent 编排层 | 消费：通过 `CON-GPC-WAES-001` 挂接 AI 授权、Agent 工具权限、模型授权、越权拦截；AI 不得直接写 GPC 平台主账 |

## 4. 平台核心能力地图

GPC 平台核心能力围绕"公共服务本体"组织：

1. **公共服务门户层**：租户门户、供应商门户、客户门户、承运商门户、监管门户、园区门户。
2. **公共服务主账层**：平台订单、需求池、供应商、客户、承运商、车辆、ASN、预约、月台、运输、POD、外部异常、跨厂异常、工厂分配、产能协同、绿色绩效、合规证据包。
3. **公共服务引擎层**：平台服务流程编排引擎（不称为"虚拟工厂运行时"）、规则与口径引擎、状态与门控引擎、证据引用引擎、AI 建议消费与治理引用引擎。
4. **公共服务连接层**：5 个根连接器在 GPC 平台侧的实例（`CON-GPC-GFIS-001` / `CON-GPC-WAES-001` / `CON-GFIS-WAES-001` 引用 / `CON-EDGE-GFIS-001` 间接 / `CON-EDGE-WAES-001` 间接）。
5. **公共服务数据层**：平台主账库、平台只读库、平台读模型、平台事件 store、平台指标 store、平台证据引用 store。
6. **公共服务开放层**：API 网关、服务目录、计量、配额、SLA、租户隔离、生态接入 SDK。

## 5. 平台内部引擎

### 5.1 平台服务流程编排引擎

- 命名：平台服务流程编排引擎 / 绿色供应链协同流程运行时 / 供应链公共服务流程引擎。
- 不允许的旧命名：虚拟工厂运行时。
- 主责：编排平台订单、需求池、供应商协作、产能协同、工厂分配、ASN、预约、运输、POD、异常协同、绿色绩效、合规证据包的公共服务流程。
- 不主责：工单释放、质量放行、库存主账、设备控制、工厂执行。
- 形式：可配置、可版本化、可执行、可留证、可复盘、可验收的 SOP 模板载体。
- 命名空间：SOP-GPC-* 模板库归属该引擎。

### 5.2 平台规则与口径引擎

- 主责：平台公共服务的业务规则、口径、可视范围、租户边界。
- 不主责：治理规则、状态门、AI 授权、证据确证（这些属于 WAES）。
- 命名空间：与 `GPC-*` 对象目录对齐，不与 `GR-*` / `GOV-*` 治理对象混用。

### 5.3 平台状态与门控引擎

- 主责：平台订单生命周期、平台异常生命周期、平台合规证据包生命周期。
- 不主责：阶段验收、状态升级结论、`partial` -> `ready_for_human_acceptance` -> `accepted` -> `complete` 的状态门（这些属于 WAES / Harness）。

### 5.4 平台证据引用引擎

- 主责：在 GPC 平台侧产生的事件、ASN、运输、POD、外部异常、合规证据包中携带 `evidenceRefs`。
- 不主责：证据的最终确证、归档、审计（这些属于 WAES）。

### 5.5 平台 AI 建议消费与治理引用引擎

- 主责：消费 AI 建议、把治理授权结果引用到平台业务事实、记录采纳与驳回。
- 不主责：发布 AI 建议、Agent 运行、模型授权、越权拦截（这些属于 XiaoC / Hermes / XGD 与 WAES）。

## 6. 平台 SOP 模板库

GPC 平台 SOP 模板以 `SOP-GPC-*` 为前缀，纳入"平台服务流程 SOP"分层，归属 GPC 平台主账：

1. 订单到交付 SOP（GPC 平台视角）
2. 需求池到分单 SOP
3. ASN 协同 SOP
4. 预约与月台 SOP
5. 运输与在途 SOP
6. POD 与签收 SOP
7. 外部异常协同 SOP
8. 跨厂异常协同 SOP
9. 工厂分配 SOP
10. 产能协同 SOP
11. 绿色绩效 SOP
12. 合规证据包 SOP
13. 生态接入 SOP
14. 公共服务开放 SOP

每个 SOP 必须显式标注主责系统 = GPC，治理确认点 = WAES，业务确认点不得在 GPC 平台侧取代 GFIS 内部确认。

## 7. 平台事件合同

GPC 平台事件以 `gpc.` 域名承载，由 GPC 主责发布；详见事件合同 §5。GPC 平台事件不得由 WAES、GFIS、PVAOS、Edge、XiaoC、Hermes/XGD 替代发布。

承载连接器：

1. `CON-GPC-GFIS-001`（GPC 平台 ↔ GFIS 业务协同）：平台订单分发、ASN/预约、运输回传、发货回执、工厂事实回流。
2. `CON-GPC-WAES-001`（GPC 平台 ↔ WAES 平台证据 / 治理状态）：平台事实证据、跨厂异常、状态升级请求、阶段验收引用。
3. `CON-GFIS-WAES-001`（GFIS ↔ WAES 工厂治理直连）：GPC 平台通过治理引用 `CON-GFIS-WAES-001` 上的工厂执行证据和异常，不直接持有该连接器。
4. `CON-EDGE-GFIS-001`（Edge ↔ GFIS 现场采集）：GPC 平台不直接消费 Edge 事件；Edge 事件先进入 GFIS。
5. `CON-EDGE-WAES-001`（Edge ↔ WAES 可选）：GPC 平台不直接消费 Edge → WAES 证据；如需消费，通过 WAES 引用。

## 8. 平台对象目录

GPC 平台主账对象以 `PO-*` / `SUP-*` / `CUS-*` / `CAR-*` / `VEH-*` / `ASN-*` / `APT-*` / `DS-*` / `SHIP-*` / `TCP-*` / `POD-*` / `EEX-*` / `FAR-*` / `FA-*` / `OM-*` / `CFE-*` 等 ID 前缀承载，详见对象目录 §4.3。

GPC 平台对象主责系统统一为 `GPC`，不与其他系统共享主账表；通过连接器和事件与 GFIS / WAES / PVAOS / Edge 协同。

## 9. 平台验收矩阵

GPC 平台验收必须同时满足以下条件：

1. **平台主账验收**：平台订单、需求池、供应商协作、产能协同、工厂分配、ASN、预约、运输、POD、异常协同、绿色绩效、合规证据包、服务开放、生态接入全部形成"主账事实 + 真实回执"。
2. **连接器验收**：5 个根连接器在 GPC 平台侧全部通过连接性、鉴权、schema、幂等、回执、证据、死信、重放、降级、越权拦截 10+ 项验证。
3. **事件验收**：平台事件链 `pvaos.organization.onboarded -> gpc.platform_order.received -> ... -> gpc.pod.signed -> waes.evidence.verified` 真实跑通。
4. **AI 边界验收**：AI 不得直接写平台主账；`ai.suggestion.governance_authorized` 不被误解为业务批准。
5. **治理验收**：WAES 通过 `CON-GPC-WAES-001` 对平台证据、状态、授权、跨厂异常形成可审计引用。
6. **工厂主账隔离验收**：GPC 平台不得绕过 GFIS 写工厂主账；GPC 平台不得接受 Edge 信号作为平台事实。

完成状态口径：

| 状态 | 判定 |
|---|---|
| `partial` | 平台主账字段完整但未真实跑通 |
| `ready_for_human_acceptance` | 平台真实跑通 + 5 根连接器验证 + 事件链验证 + 治理引用验证 + 工厂主账隔离验证通过，待人工确认 |
| `complete` | 完成真实运行态联调、业务证据和人工验收 |

## 10. 平台建设路线

GPC 平台建设路线分四阶段：

1. **S1 平台本体骨架**：平台主账对象、平台 SOP 模板、平台服务流程编排引擎、5 根连接器中 `CON-GPC-GFIS-001` / `CON-GPC-WAES-001` 实例。
2. **S2 公共服务接入**：客户、供应商、承运商、监管、园区接入与上下文；PVAOS 入口绑定。
3. **S3 工厂协同闭环**：与 GFIS 协同跑通订单到交付主线；工厂分配、跨厂异常、合规证据包上线。
4. **S4 治理与 AI 编排接入**：WAES 治理引用全面上线；AI 建议消费、治理授权、越权拦截上线。

每一阶段都必须以"平台主账真实回执"为完成依据，配置完成、UI 完成、连通性完成均不得视为阶段完成。

## 11. 一句话口径

GPC 绿色供应链公共服务平台本体 = 客户/供应商/承运商/监管/园区与工厂之间开展绿色供应链协同的公共服务场所；它主责 12 类平台公共服务主账事实，承担 3 类公共服务支撑能力，不主责工厂执行、租户/门户、治理审批、Edge 现场采集、知识主存和 AI 编排；它通过 5 个根连接器与 GFIS、WAES、PVAOS、Edge、Brain、XiaoC·Hermes·XGD 协同，但任何系统不得越权写入 GPC 平台主账。
