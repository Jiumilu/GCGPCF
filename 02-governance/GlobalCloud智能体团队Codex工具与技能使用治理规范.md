# GlobalCloud智能体团队Codex工具与技能使用治理规范

日期：2026-06-08  
状态：Codex 工具与技能治理规范 v1

## 1. 目的

本规范用于统一控制：

1. Codex 主线程与子线程
2. Shell
3. Git
4. Linear
5. Browser / Computer Use
6. Skill
7. 自动化
8. 凭据与 API key

## 2. 工具分层

### 2.1 总控级工具

1. Codex 主线程
2. 文档编辑
3. Linear 项目台账
4. 周报与控制塔

### 2.2 专项执行级工具

1. Shell
2. Git
3. rg / sed / ls / find
4. 仓库级只读预检

### 2.3 运行验证级工具

1. Browser
2. Computer Use
3. 本地运行命令
4. 验证脚本

## 3. 工具使用矩阵

| 工具 | 设计控制环境 | 项目仓库环境 | 本地运行环境 | 集成验证环境 | 默认权限 |
|---|---|---|---|---|---|
| Codex 主线程 | 是 | 否 | 否 | 否 | 汇总与控制 |
| Codex 专项线程 | 是 | 是 | 是 | 是 | 受专项边界限制 |
| Shell | 只读为主 | 只读为主 | 可运行 | 可运行 | 逐阶段放开 |
| Git | 否 | 是 | 是 | 是 | 默认只读 |
| Linear | 是 | 否 | 否 | 否 | 只用于项目控制 |
| Browser | 否 | 否 | 是 | 是 | 运行验证 |
| Computer Use | 否 | 否 | 是 | 是 | 运行验证 |
| Skill | 是 | 是 | 是 | 是 | 按任务匹配加载 |
| 自动化 | 是 | 否 | 否 | 否 | 需可停、可删、可审计 |

## 4. Skill 使用规则

1. 任务明确匹配 skill 时必须使用对应 skill。
2. 只加载当前任务所需 skill，不批量加载。
3. skill 的输出必须服从当前阶段环境门。
4. skill 不能绕过仓库 preflight、证据门和状态门。

## 5. 自动化规则

1. 自动化必须有明确目的。
2. 自动化必须能删除。
3. 自动化必须避免长期重复无效输出。
4. 阶段结束后，旧自动化必须停用或删除。

## 6. 凭据与密钥治理

1. API key 不写入工作区文档。
2. API key 不回填到普通台账。
3. 使用外部 API 时，只记录动作和结果，不回显密钥本体。
4. 临时集成密钥优先于长期个人密钥。

## 7. Codex 线程治理

1. 小即主线程负责总控。
2. 专项线程负责责任域执行。
3. 状态升级只能由小即汇总后发布。
4. 专项线程不得自行把状态升级为 `complete`。

## 8. 当前适用判断

当前阶段适用的工具治理重点：

1. 项目仓库层继续以只读预检和运行前样本定位为主
2. Browser / Computer Use 暂未系统展开
3. Linear 继续只做台账控制，不做完成状态拔高
4. 任何工具输出都不得越过环境门和状态门
