# GlobalCloud 绿色供应链体系设计-实现追踪矩阵规范

日期：2026-06-08  
状态：设计-实现追踪矩阵规范 v1  
用途：保证设计条目能够追踪到真实仓库、模块、文件、交付物和验收场景。

## 1. 追踪原则

任何实现都必须能回答：

1. 它满足哪条设计
2. 它属于哪个交付包
3. 它落在哪个仓库和模块
4. 它对应哪个对象或事件
5. 它如何被验证

## 2. 最小追踪字段

每条追踪项必须至少包含：

1. 设计来源文档
2. 设计条目编号或章节
3. 交付包
4. 专项负责人
5. 仓库路径
6. 模块/文件落点
7. 对象/事件关联
8. 验收场景关联
9. 当前状态

## 3. 追踪层次

至少建立三类追踪：

1. 设计 -> 仓库/文件
2. 对象/事件 -> 代码/接口
3. SOP/验收场景 -> 功能落点/验证结果

## 4. 执行要求

1. 进入真实仓库实施前，先建立追踪项。
2. 任何代码变更若无法挂到追踪项，不得视为体系内有效交付。
3. 任何验收结论若无法回指到追踪项，不得视为设计达成。

## 5. 团队责任

1. 各专项负责人建立本域追踪项。
2. 小即负责跨仓库、跨专项追踪的一致性。
3. 评衡与证验负责检查追踪链是否闭合。
