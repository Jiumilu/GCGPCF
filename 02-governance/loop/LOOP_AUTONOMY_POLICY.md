# Loop Autonomy Policy

## 自动化等级

| 等级 | 名称 | 定义 | 默认状态 |
|---|---|---|---|
| L0 | 观察模式 | 只读分析、报告、建议，不改文件 | 默认可用 |
| L1 | 文档治理模式 | 新增/修订受控文档、台账、KDS 本地镜像、模板 | 用户授权后可用 |
| L2 | 开发闭环模式 | 小范围本地代码、测试、mock、dry-run、validator、evidence | 用户授权后可用 |
| L3 | 托管冲刺模式 | 在受控边界内连续推进多轮低风险任务 | 必须用户明确说“启动 Loop L3 托管冲刺模式” |
| L3.5 | 受控真实接口验证模式 | 在白名单、可回滚、可审计边界内验证真实 API 读、真实鉴权、validate-only/sandbox/staging/单条测试写入 | 必须用户明确说“启动 Loop L3.5 受控真实接口验证模式” |
| L4 | 全自动运营模式 | 自动推进开发、治理、测试、文档、提交、非主分支推送和 KDS dry-run/白名单同步 | 可执行但未激活；必须用户明确说“启动 Loop L4 全自动运营模式” |
| L5 | 完全生产自治模式 | 在强授权生产治理宪法内自主发现、修复、发布、监控、回滚和复盘 | 可执行但未激活；必须使用 L5 强授权口令 |

## L3 托管冲刺规则

当用户明确说“启动 Loop L3 托管冲刺模式”时，Loop 可在受控边界内连续推进最多 15 轮或 2 小时，以先到者为准。

| 约束 | 规则 |
|---|---|
| 轮次数上限 | 最多 15 轮 |
| 时间上限 | 最多 2 小时 |
| 状态上限 | 不得自动超过 `partial`、`loop_running` 或 `evidence_ready`，具体受门禁约束 |
| 人工确认 | 触及本政策人工确认项时立即暂停 |
| evidence | 每轮必须生成或更新 evidence |
| 控制板 | 每轮结束必须更新 `LOOP_CONTROL_BOARD.md` 或登记未更新原因 |
| 默认继续 | 未触发停止条件且任务队列不为空时，必须自动进入下一轮；阶段性汇报、完成单轮、完成两轮或完成一个小任务不构成停止条件 |
| 停止说明 | 若 L3 停止，必须记录停止类型、停止证据、是否符合 L3 停止规则、剩余轮次和已用时间 |

## L3 连续推进与停止规则

L3 启动后，Loop 的默认行为是持续推进，而不是阶段性收口。阶段性汇报只能作为 evidence 或进度报告，不得替代下一轮执行。

### 允许停止条件

L3 只能在满足以下任一条件时停止：

1. 已连续完成 15 轮。
2. 已运行满 2 小时。
3. 触发 P0/P1 硬停止条件。
4. 用户明确说暂停、停止、收口、只汇报或退出 L3。
5. 当前任务队列为空，且已生成新的候选任务但需要用户确认。

### 非停止条件

以下情况不得作为 L3 停止理由：

- 完成 1 轮或 2 轮。
- 完成一个小任务。
- 做了阶段性收口汇报。
- 生成了下一轮建议。
- 当前状态仍为 `partial`。
- KDS TOKEN 暂缓但本轮不需要真实 KDS API。

### L3 轮次预算计数器

每轮结束时必须记录：

| 字段 | 要求 |
|---|---|
| L3 session | active / stopped |
| 已完成轮次 | `n/15` |
| 剩余轮次 | `15-n` |
| 已用时间 | `x/120 分钟` |
| 停止类型 | none / hard_stop / user_stop / budget_exhausted / time_exhausted / task_queue_empty |
| 停止证据 | 未停止时写 `无` |
| 下一轮 | 未停止时必须给出下一轮 Round ID 或任务 |

## L3.5 受控真实接口验证规则

L3.5 用于补齐 L3 dry-run 与真实系统之间的验证缺口。它不是全自动运营，也不是生产自治；它只允许在显式授权下执行有限、可回滚、可审计的真实接口验证。

| 约束 | 规则 |
|---|---|
| 启动口令 | 必须明确说“启动 Loop L3.5 受控真实接口验证模式” |
| 运行上限 | 最多 5 轮或 1 小时，以先到者为准 |
| 授权字段 | 必须包含授权项目、授权环境、授权 API、时间窗、允许动作、禁止动作、回滚策略和验收指标 |
| 允许真实动作 | 真实 API 读、真实鉴权、validate-only、dry-run、sandbox/staging 写入、单条白名单可回滚测试写入 |
| 禁止真实动作 | 批量生产写入、删除、权限变更、Token/密钥变更、生产部署、不可回滚写入 |
| 证据要求 | 记录 endpoint、请求摘要、状态码、响应结构、时间戳、回滚/清理结果；不得记录密钥或 TOKEN |

完整规则见 `02-governance/loop/LOOP_L3_5_REAL_API_VERIFICATION.md`。

## L4 全自动运营规则

L4 是项目群开发运营自动化能力，状态为 `executable / not activated`。未出现明确 L4 启动口令时，不得由 L3 或 L3.5 自动升级。

| 约束 | 规则 |
|---|---|
| 启动口令 | 必须明确说“启动 Loop L4 全自动运营模式” |
| 运行上限 | 初始最多 30 轮或 4 小时，以先到者为准 |
| 自动任务选择 | 允许读取任务队列并自动选择下一轮目标 |
| Git | 允许本地 commit；允许白名单非主分支 push；禁止直接改 main、合并 PR |
| KDS | 允许 KDS dry-run；白名单下允许开发空间同步；不得写入或暴露 TOKEN |
| 生产 | 默认禁止生产部署、真实生产写入、生产权限变更 |

完整规则见 `02-governance/loop/LOOP_L4_AUTONOMOUS_OPERATIONS.md`。

## L5 完全生产自治规则

L5 是最高等级生产自治能力，状态为 `executable / not activated`。L5 的“完全”指在事先授权的生产治理宪法内完成生产问题发现、修复、发布、监控、回滚和复盘，不代表无边界自治。

| 约束 | 规则 |
|---|---|
| 启动口令 | 必须使用包含项目、环境、分支、时间窗、任务范围、允许动作、禁止动作、回滚策略、验收指标和监控指标的强授权口令 |
| 运行上限 | 初始最多 10 轮或 2 小时，以先到者为准 |
| 生产动作 | 在授权范围内可执行生产读写、部署、监控、回滚 |
| 不可突破底线 | 不得扩大自身权限、修改最高治理规则、关闭监控、删除审计证据、隐藏失败、越过授权边界 |
| 最高原则 | 生产连续性 > 数据安全 > 客户影响最小化 > 可回滚 > 可审计 > 自动化速度 |

完整规则见 `02-governance/loop/LOOP_L5_FULL_PRODUCTION_AUTONOMY.md`。

| 更新 KDS 本地镜像 | 否 | 是 | 是 | 是 | 是 | 是 | 是 |

## 各等级禁止动作

| 动作 | 规则 |
|---|---|
| Git push | 所有等级默认禁止，必须用户明确授权 |
| 合并主分支 | 所有等级禁止自动执行 |
| 删除文件或大规模迁移 | 所有等级禁止自动执行 |
| 生产配置修改 | 所有等级禁止自动执行 |
| 真实 API 写入 | L0-L3 禁止；L3.5/L4/L5 仅在对应专项政策与显式授权内允许 |
| 真实 KDS TOKEN 写入 | 所有等级禁止自动执行 |
| 权限变更 | 所有等级禁止自动执行 |
| 部署 | L0-L4 默认禁止；L5 仅在强授权内允许 |
| 标记 `accepted` / `integrated` | 所有等级禁止自动执行 |
| 改变项目战略定位或架构主结论 | 所有等级禁止自动执行 |

## 必须人工确认的动作

- Git push、创建 PR、合并分支。
- 删除、重命名或大规模迁移文件。
- 写入生产、外部系统或真实业务数据；L3.5/L4/L5 仅可在对应专项政策和显式授权内执行。
- 使用或写入真实 KDS TOKEN。
- 执行 `bench migrate`、运行态 schema sync 或运行态写 API。
- 修改项目战略定位、架构主结论、验收口径。
- 将项目状态升级到 `accepted` 或 `integrated`。
- 需要用户做业务判断、取舍或签收。

## 自动暂停条件

| 条件 | 动作 |
|---|---|
| 出现 P0 风险 | 立即暂停，写入风险记录 |
| 测试失败且无法局部修复 | 暂停并记录失败命令和最小复现 |
| Git 冲突或 dirty 风险不可判断 | 暂停并报告 |
| 发现密钥、TOKEN、证书或权限问题 | 暂停并保护敏感信息 |
| 触及未授权生产写入或真实外部写入 | 暂停并请求确认 |
| 涉及删除/迁移 | 暂停并请求确认 |
| 需要业务判断 | 暂停并请求确认 |
| 文档与事实严重冲突 | 暂停并登记污染风险 |

## Git commit/push 策略

- 默认不自动 commit，不自动 push。
- L3 可本地 commit 的前提：
  - 用户明确允许 L3 本地 commit。
  - `git diff --check` 通过。
  - 无敏感文件。
  - 变更范围与本轮任务一致。
  - commit message 包含 Round ID。
- push 必须另行取得用户明确授权。

## KDS TOKEN 策略

- 本开发机阶段 KDS TOKEN 可暂缓。
- TOKEN 缺失时允许本地 KDS 镜像与台账治理，但不得声明真实 KDS API 双向同步完成。
- 不得把 TOKEN 写入 Git、`.kds/`、日志、文档或 evidence。

## 外部 API 边界

| 类型 | 自动化规则 |
|---|---|
| mock API | L2/L3 可执行 |
| dry-run API | L2/L3 可执行，必须可清理、可复现 |
| 真实写 API | L0-L3 必须人工确认；L3.5/L4/L5 按专项政策与显式授权执行 |
| 生产 API | L0-L4 必须人工确认；L5 按强授权执行 |
| 资金、权限、部署相关 API | 默认禁止自动执行 |
