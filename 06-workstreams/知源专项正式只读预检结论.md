## 2. Git 与工作区事实

## 3. 责任边界符合度

按当前仓库抽样，KDS 当前符合“知识主存”定位；Brain 负担知识引擎治理域：

1. 仓库明确保留 `Markdown + Git` 为 system of record 的口径。
2. `GBRAIN_MIGRATION.md` 明确 Brain 是 retrieval and graph index layer，不是唯一知识真源。
3. `README.md`、`RAG_ADMISSION_RULES.md`、`wiki_trust_audit.py`、`rag_admission_policy.py` 共同支持“知识准入、准入过滤、引用回指、可信化审计”的治理方向。
4. 当前抽样结果不支持把 Brain 解释为业务主账系统。

## 4. 当前已确认的仓库级证据

已确认的关键证据包括：

1. `README.md`
2. `GBRAIN_MIGRATION.md`
3. `RAG_ADMISSION_RULES.md`
4. `_governance/scripts/rag_admission_policy.py`
5. `_governance/scripts/wiki_trust_audit.py`
6. `docs/harness/acceptance.md`
7. `docs/standardization/standardization-audit-2026-06-06.md`

这些证据能证明：

1. KDS 当前确实承担知识主存治理；Brain 承担 RAG 准入规则和检索层约束。
2. 当前已有 Brain migration、doctor、search、trust audit 等运行前工具链证据。
3. 当前知识域不是纯概念设计，而是有真实仓库级实现与治理脚本支撑。

这些证据还不能证明：

1. 知识发布链已经完全健康
2. ingest / 引用回指 / 失效拦截已经全部闭环
3. Brain doctor 当前已健康
4. remote 已绑定并可作为远程治理证据

## 5. 预检结论

1. 知源专项已完成 Brain 仓库级第一轮正式只读预检。
2. KDS 当前仍可被认定为知识主存与知识引擎治理域的核心仓库。
3. Brain 与 “Markdown 为真源、Brain 为检索层” 的当前架构口径一致。

1. 尚未补齐 Brain remote 证据。

3. 尚未关闭 Brain doctor / schema / timeline 等运行阻塞。
