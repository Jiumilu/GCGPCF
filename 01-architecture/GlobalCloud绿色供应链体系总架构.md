# GlobalCloud 绿色供应链体系总架构

日期：2026-06-10
状态：架构口径修正稿 v2（统一基线）
适用范围：当前工作区全部绿色供应链体系设计文档、专项图、厂行专项、宪衡专项、链同专项、知源专项、灵策/评证/证验专项、Codex 智能体团队、连接器合同、事件合同、对象目录。

本文是体系级统一基线；本文之后，体系内全部文档应统一引用本文件定义的总定位、主账边界、链路、连接器编号与 SOP 分层。

## 1. 总定位

GlobalCloud GPCF 是面向绿色供应链的公共服务与治理体系，不是单一软件产品，也不是某一套 ERP 的二开。

体系由六类主责系统组成，每一类系统都有清晰的主账边界，任何系统不得越权写入其他系统的主账事实：

| 系统 | 在体系中的身份 | 主账/主责 | 不应承担 |
|---|---|---|---|
| GPC | 绿色供应链公共服务平台本体，平台主线与公共服务平台本体 | 平台订单、需求池、样品申请、客户签样、转量产门禁、供应商协作、产能协同、工厂分配、ASN、预约、运输、POD、异常协同、绿色绩效、合规证据包、服务开放、生态接入 | 工厂主账、治理审批主账、租户与门户主账、现场采集主账 |
| PVAOS | 平台运营、租户、组织、伙伴、用户、权限、门户、项目和应用入口底座 | 租户、组织、用户、角色、权限、伙伴档案、项目空间、应用入口、门户菜单、平台运营配置 | 平台订单主账、工厂事实主账、治理审批主账 |
| GFIS | 工厂执行系统 / 工厂事实主账 | 配方研发、样品打样、样品检测、工厂订单、工单、生产报工、质量检验、质量放行、库存事务、批次、设备维护、发货出库、工厂执行证据 | 平台订单主账、客户签样主账、转量产主账、租户/门户主账、治理审批主账、现场信号主账 |
| WAES | 治理、证据确证、状态门控、合规、风险、审计、AI 越权控制系统 | 治理规则、证据确证、状态门控、合规校验、风险控制、审计追踪、模型授权、AI 越权拦截、异常升级、验收证据管理 | 平台订单主账、工厂执行主账、租户/门户主账 |
| Edge | 现场采集与边缘缓冲层，优先服务 GFIS 的工厂执行事实确认 | 设备信号采集、协议转换、断网缓存、补传、去重、报警回执 | 业务主账、治理审批、平台订单、租户/门户 |
| KDS | 企业知识主存（唯一数据源） | 知识真源、版本、发布、权限、归档 | 业务事实、AI 直接建议 |
| Brain / LLM Wiki | 知识编制与引擎候选 | 编制视图、引擎索引、消费服务 | 治理审批、反写主存 |
| XiaoC（蚁后） / Hermes / XGD（大象） | AI 与 Agent 编排层 | XiaoC 负责 Prompt、MCP、模型路由、任务拆解和结果汇总；Hermes/XGD 负责长程 Agent、重分析、多端交互和运行回执 | 直接写业务事实、绕过治理授权、越权调用工具 |

仓库中已有的 gcfis_custom、ERPNext 相关代码只能称为 GFIS 历史实现资产、适配器资产或 legacy reference，不作为 GFIS 主线定义来源；GFIS 主线定义以本文、对象目录、事件合同、连接器合同、厂行专项为准。

## 2. 主账边界

体系明确禁止任何系统越权写入其他系统的主账事实：

1. GPC 主账：平台订单、需求池、样品申请、客户签样、转量产门禁、供应商协作、产能协同、工厂分配、ASN、预约、运输、POD、异常协同、绿色绩效、合规证据包、服务开放、生态接入。
2. GFIS 主账：配方研发、样品打样、样品检测、工厂订单、工单、生产报工、质量检验、质量放行、库存事务、批次、设备维护、发货出库、工厂执行证据。
3. WAES 主账：治理规则、证据确证、状态门控、合规校验、风险控制、审计追踪、模型授权、AI 越权拦截、异常升级、验收证据管理。
4. PVAOS 主账：租户、组织、用户、角色、权限、伙伴档案、项目空间、应用入口、门户菜单、平台运营配置。
5. Edge 缓冲账：设备信号、报警、缓存事件、补传事件；Edge 不得在自身数据库落地业务主账事实，必须在 GFIS 规则中确证后才进入业务流。
6. Brain/LLM Wiki 知识账：知识文档、版本、发布、权限；不得承接业务主账事实。
7. XiaoC（蚁后）/Hermes/XGD（大象） AI 账：Agent 运行、建议、越权拦截记录；不得承接业务主账事实。

## 3. 主链路

体系只承认以下三条主链路，其他任何"主链"都属于实现细节或局部视图。

### 3.1 平台业务协同链路

```text
PVAOS → GPC → GFIS → GPC → 外部生态服务 / WAES
```

1. PVAOS 提供租户、组织、用户、角色、伙伴和门户入口上下文。
2. GPC 接收并分发平台订单、样品申请、客户签样、转量产、ASN、预约、运输、POD、跨厂协同。
3. GFIS 接收已转量产订单、形成配方研发/样品打样/样品检测、工单/质量/库存/批次/发货事实并回传。
4. GPC 收齐 GFIS 事实后，形成平台层最终事实（发货、运输、POD、签收、异常）。
5. 外部生态服务（客户/供应商/承运商/银行/监管）经由 GPC 接入。
6. WAES 在上述每一步只负责证据、治理、状态、授权和合规校验，不参与事实写入。

### 3.2 工厂执行事实链路

```text
Edge → GFIS → GPC
GFIS → WAES
```

1. Edge 采集设备信号、报警、断网缓存、补传；优先服务 GFIS。
2. GFIS 将信号转成工单、质量、库存、批次、设备维护、发货出库事实。
3. GFIS 通过连接器将工厂事实回传给 GPC。
4. GFIS 同时向 WAES 推送工厂证据、状态和异常。

### 3.3 治理与证据链路

```text
GPC → WAES
GFIS → WAES
Edge → WAES
AI Agent → WAES
Brain → WAES
```

1. 任何系统产生的证据、状态、异常、授权请求都必须可被 WAES 引用、确证、驳回和审计。
2. AI Agent 在调用任何工具或模型前必须先经 WAES 授权；越权动作由 WAES 拦截。
3. Brain 在发布、引用、导出知识前必须经 WAES 治理校验。

## 4. 连接器清单

体系只承认以下 5 个跨系统连接器编号与定位；其他连接器属于实现层扩展，必须挂接到这 5 个根连接器下。

| 编号 | 名称 | 两端 | 等级 | 用途 | 默认写权限 |
|---|---|---|---|---|---|
| CON-GPC-GFIS-001 | GPC 与 GFIS 的业务协同连接器 | GPC ↔ GFIS | C1/C2 | 平台订单分发、ASN/预约、运输回传、发货回执、工厂事实回流 | 业务主账前置接口；不绕过业务确认 |
| CON-GPC-WAES-001 | GPC 与 WAES 的平台证据 / 治理状态连接器 | GPC ↔ WAES | C1/C4 | 平台事实证据、跨厂异常、状态升级请求、阶段验收引用 | 仅证据与治理引用，不写业务主账 |
| CON-GFIS-WAES-001 | GFIS 与 WAES 的工厂治理 / 异常 / 审计 / 证据直连连接器 | GFIS ↔ WAES | C1/C4 | 工厂执行证据、工厂异常、审计追踪、证据确证 | 仅证据与治理引用，不写业务主账 |
| CON-EDGE-GFIS-001 | Edge 与 GFIS 的现场采集连接器 | Edge → GFIS | C1 | 设备信号、报警、缓存补传、回执 | Edge 不在自身数据库落地业务主账 |
| CON-EDGE-WAES-001 | Edge 与 WAES 的异常 / 安全 / 审计证据连接器（可选） | Edge → WAES | C1/C4 | 边缘异常、节点安全、审计证据；不替代 CON-EDGE-GFIS-001 | 可选；存在时不得绕过 GFIS 写业务主账 |

明确：

1. `CON-GFIS-WAES-001` 不废弃，作为 GFIS ↔ WAES 工厂治理直连连接器保留。
2. 任何"绕过 GFIS 写业务主账"的连接器在体系中不得成立。
3. 任何"绕过 GPC 写平台主账"的连接器在体系中不得成立。
4. 任何"绕过 WAES 改写治理状态"的连接器在体系中不得成立。

## 5. SOP 分层

SOP 模板必须显式标注所属分层，不允许跨层混用：

| 分层 | 主责系统 | 模板 ID 前缀 | 说明 |
|---|---|---|---|
| 平台服务流程 SOP | GPC | `SOP-GPC-*` | 平台订单、样品申请、客户签样、转量产、ASN、预约、运输、POD、跨厂协同、外部生态服务 |
| 工厂执行 SOP | GFIS | `SOP-GFIS-*` | 配方研发、样品打样、样品检测、工厂订单接收、工单、齐套、质量、库存、批次、设备维护、发货出库 |
| 治理规则 SOP | WAES | `SOP-WAES-*` | 规则生效、指标口径、状态升级、证据确证、阶段验收、连接器治理、知识准入 |
| 知识型 SOP | Brain / LLM Wiki | `SOP-KNOW-*` | 知识发布、版本、引用、AI 可见范围；不承接业务事实 |
| 平台运营 SOP | PVAOS | `SOP-PVAOS-*` | 租户、组织、伙伴、用户、角色、门户 |
| 边缘接入 SOP | Edge | `SOP-EDGE-*` | 采集、协议转换、缓存、补传、去重、回执；不承接业务事实 |

## 6. 引擎命名修正

GPC 的内部流程引擎统一命名为"平台服务流程编排引擎 / 绿色供应链协同流程运行时 / 供应链公共服务流程引擎"，不得再称为"虚拟工厂运行时"。"虚拟工厂"概念只用于 GFIS 内部的工厂执行模型，不属于 GPC 平台本体。

## 8. 一句话口径

GlobalCloud GPCF 是面向绿色供应链的公共服务与治理体系；GPC 是绿色供应链公共服务平台本体；PVAOS 提供平台运营与门户底座；GFIS 负责工厂执行与工厂事实主账；WAES 负责治理、证据确证、状态门控与风险控制；Edge 负责现场采集与边缘回传；KDS 负责知识主存；Brain / LLM Wiki 负责知识编制与引擎；XiaoC（蚁后）负责 AI 能力生产、Prompt/MCP、模型路由、任务拆解和结果汇总；Hermes/XGD（大象）负责长程 Agent 承载、重分析、多端交互和运行回执。各系统通过对象主责、事件合同、连接器合同和证据链路协同，但任何系统不得越权写入其他系统的主账事实。
