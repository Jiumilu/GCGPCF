# GlobalCloud 绿色供应链体系界面实施差距清单

日期：2026-06-08
状态：界面实施差距清单 v1
用途：把当前已形成的界面分阶段治理规则、体验骨架规范、最终交付界面规范、对话模式与操作模式规范、统一组件与设计令牌规范，转成实际实施差距、优先级和关闭路径。

---

## 1. 评估口径

本清单按五类规范检查当前体系界面实施差距：

1. 分阶段治理差距
2. 页面骨架差距
3. 对话与操作差距
4. 样式与组件差距
5. 多端一致性差距

差距分级：

- `P0`：不解决就无法形成最小闭环或会直接造成拼凑式体验
- `P1`：不解决会导致正式交付阶段体验明显不一致
- `P2`：可延后到统一收口阶段处理

---

## 2. 当前差距总览

### 2.1 P0 差距

1. **平台订单界面主线未收口**
   - 当前问题：`GPC` 现有原型不能作为正式主线持续扩展，平台订单最终界面结构无法稳定。
   - 影响：订单协同页、订单详情页、订单异常页无法形成长期统一体验。
   - 建议：按 `GPC` 路线重建。

2. **签收界面主线未收口**
   - 当前问题：签收相关能力仍绑定在历史原型思路上，无法保证正式交付的一致性。
   - 影响：POD、签收确认、签收异常、证据回执体验会分裂。
   - 建议：按 `GPC` 正式主线重建。

3. **异常处理界面缺统一异常壳层**
   - 当前问题：异常处理容易分散到不同系统各自实现。
   - 影响：用户会感知为多个产品，各自有不同异常关闭逻辑。
   - 建议：建立统一异常页骨架和统一异常关闭模式。

4. **AI 建议展示仍缺统一组件收口**
   - 当前问题：虽然规范已定义，但当前尚无统一 AI 建议卡片、引用区、风险提示条的组件实现基线。
   - 影响：AI 在不同产品中的展示方式会发散。
   - 建议：优先纳入组件库首批建设。

### 2.2 P1 差距

1. **质量页仍需二次开发后再统一体验**
2. **项目初始化与发货页目前偏“产品内实现”，但还未完成最终交付层统一收口**
3. **证据/审计页缺显式统一组件标准**
4. **治理确认页缺统一工作台实现模板**
5. **工作台类页面尚未收口成同一右侧上下文/证据/AI 辅助结构**

### 2.3 P2 差距

1. 多端视觉细节统一尚未开始
2. 统一图标系统尚未落地
3. 统一主题切换与品牌层策略尚未落地
4. 统一空态/加载态/错误态文案库尚未落地

---

## 3. 分系统差距

### 3.1 WAES

当前优势：

- 治理与控制塔结构相对集中
- 证据、状态、指标、治理确认逻辑边界清晰

主要差距：

1. 治理确认类页面需统一工作台模板
2. 证据确证、规则生效、状态升级页需统一危险操作确认模式
3. AI 授权与治理提示区需统一为治理型对话/说明组件

### 3.2 PVAOS

当前优势：

- 组织、伙伴、项目属于较容易统一的对象入口层

主要差距：

1. 项目初始化页需与最终配置页骨架对齐
2. 组织/伙伴详情页需与全局详情骨架统一
3. 入口层不应长成独立风格门户

### 3.3 GPC / GPC

当前优势：

- 协同对象清晰：平台订单、样品申请、客户签样、转量产、ASN、预约、运输、POD、异常

主要差距：

1. 平台订单必须重建正式主线
2. 样品确认必须建立独立状态、证据和转量产门禁
3. 签收必须重建正式主线
4. 异常处理必须统一壳层
5. 运输与 POD 页需统一证据/回执结构

### 3.4 GFIS

当前优势：

- 配方研发、样品打样、工厂订单、工单、库存、批次等对象适合收敛到统一详情与工作台结构

主要差距：

1. 质量页二次开发后需统一到最终骨架
2. 线边、设备、执行工作台后续需要按统一工作台模式建设
3. 与 Edge 关联的现场事实展示需保持业务事实与设备信号分层

### 3.5 Brain / XiaoC / Hermes / XGD

当前优势：

- AI 与知识域可单独做统一交互系统

主要差距：

1. AI 对话页与 AI 辅助侧栏尚无统一实际组件体系
2. 知识引用区、证据引用区、来源标识区需统一
3. 多端 AI 交互语义需统一

---

## 4. 差距关闭顺序

### 第一批必须先关

1. 平台订单主线界面收口路径
2. 签收主线界面收口路径
3. 异常页统一壳层
4. AI 建议统一组件首批定义

### 第二批再关

1. 证据/审计页统一
2. 治理确认页统一
3. 质量页统一
4. 工作台统一

### 第三批最后关

1. 多端视觉细节统一
2. 图标系统统一
3. 空态/加载态/错误态文案系统统一

---

## 5. 当前阶段结论

当前不是缺规范，而是已经进入：

**规范齐全，执行收口不足**

也就是说，下一阶段重点不应再新增同层界面规范，而应转入：

1. 界面差距关闭
2. 统一组件库建设
3. 高风险模块先行收口
