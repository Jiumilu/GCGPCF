# Studio操作工作台验证方案

日期：2026-06-22  
状态：操作工作台验证方案 v1  
定位：`WAES -> Studio` 首条正式实施主线的第三控制成果

## 1. 验证目的

本方案用于验证一个明确命题：

`WAES` 母框架在补齐 P0/P1 关键项后，是否能够被 `Studio` 作为“面向个人、团队、以知识系统为基础的智能专业工作台”稳定继承。

本方案不做以下事情：

1. 不把 `Studio` 全项目所有页面都纳入本轮。
2. 不把 `Studio` 验证写成 `WAES` 母框架已经成功扩散。
3. 不跳过母框架补差，直接做孤立 `Studio` 新设计。

## 2. 验证对象

本轮验证对象固定为：

1. 项目：`Studio`
2. 页面类：`操作工作台页`
3. 主目标：知识驱动协作
4. 次目标：工程与流程工作台能力
5. 次目标：AI 增强工作台能力

## 3. 验证命题

本轮必须验证以下 5 个命题：

1. `Studio` 是否能继承 `WAES` 的操作工作台骨架，而不丢失自身知识驱动特性。
2. `Studio` 是否能继承 `WAES` 的状态、证据、审计语义，而不退化为普通工具页。
3. `Studio` 是否能在同一工作台中容纳知识上下文、流程推进和 AI 建议边界。
4. `Studio` 是否需要局部例外；如果需要，例外是否可控。
5. `Studio` 的继承结果是否足以成为 `PKC / XGD` 后续扩散前置验证。

## 4. 验证范围

### 4.1 必须覆盖的区块

`Studio` 操作工作台页至少必须显式覆盖：

1. 主任务/主对象区
2. 状态区
3. 风险与阻断区
4. 证据与引用区
5. 审计与回执区
6. AI 建议与人工接管区

### 4.2 允许项目级差异的部分

`Studio` 可保留差异的部分：

1. 知识对象卡片的具体排版
2. 工程流程面板的具体内容组织
3. AI 辅助外观细节
4. 工程工具场景下的布局密度

但这些差异不得突破：

1. `WAES` 母框架骨架边界
2. 统一状态语义
3. 统一证据与审计语义
4. AI 建议与业务事实分离规则

## 5. 保留层与可变层

### 5.1 保留层

以下内容必须继承 `WAES` 母框架，不得自行重写成另一套：

1. 工作台总骨架
2. 状态区逻辑
3. 证据/引用/审计关系
4. AI 建议区与人工接管区边界
5. 危险操作确认规则

### 5.2 可变层

以下内容允许按 `Studio` 场景调整：

1. 知识上下文面板内容
2. 工程流程信息层次
3. 双栏或三栏分屏比例
4. AI 辅助区默认展开策略

## 6. 验证方式

### 6.1 结构验证

验证 `Studio` 页面骨架是否仍属于项目群定义的 `操作工作台页`，而不是退回到工具页拼装。

### 6.2 语义验证

验证以下语义是否仍保持 `WAES` 母框架口径：

1. 状态
2. 风险
3. 证据
4. 审计
5. AI 建议

### 6.3 交互验证

验证以下交互是否成立：

1. 主操作与次操作分层
2. 危险操作二次确认
3. 引用与证据可回指
4. AI 建议可采纳、可驳回、可人工接管
5. 长文本、窄屏、分屏切换时结构不崩

### 6.4 例外验证

如 `Studio` 需要偏离 `WAES` 母框架，必须：

1. 提交例外编号
2. 明确不适配层
3. 明确替代方案
4. 在 Loop round 中显式记录

## 7. Loop 验证要求

### 7.1 必填字段

涉及本验证方案的 round 必须写出：

1. `UI scope = true`
2. `UI gate status`
3. `Surface = operation-workbench`
4. `Tools used`
5. `Verification`
6. `Status ceiling`
7. 如有例外，则写 `exception_id` 等例外字段

### 7.2 关键门禁要求

本验证方案的关键门禁固定为：

1. `G1 Surface Structure`
2. `G4 Evidence And Governance`
3. `G5 AI Fact Separation`
4. `G8 Runtime Verification`
5. `G9 Scope Control`

任何一项失败，本轮不得宣称验证通过。

## 8. 通过标准

`Studio` 操作工作台验证通过，至少必须同时满足：

1. 页面结构仍清晰属于 `操作工作台页`
2. `WAES` 母框架的状态、证据、AI 边界未被打散
3. 知识驱动协作能力已真实映射到主工作台而不是外挂侧页
4. 工程与流程工作台能力未破坏主骨架
5. AI 增强能力未伪装成系统事实
6. 至少形成一个真实 `UI gate` round

## 9. 未通过判定

以下任一情况出现，即视为本轮未通过：

1. `Studio` 自行长出第二套工作台骨架
2. 证据/引用/审计关系被压缩为普通说明区
3. AI 建议区与系统事实区混写
4. 为适配 `Studio` 而把 `WAES` 母框架打散到不可复用
5. 未做真实验证，只给出设计性描述

## 10. 通过后的影响

若本验证通过，只能得出以下结论：

1. `WAES -> Studio` 首条实施主线成立
2. `PKC` 可以进入下一轮继承评估
3. `XGD` 可以开始受控继承评估

仍不得直接声明：

1. `WAES` 已成为项目群基线
2. `Studio` 已完成全项目 UI 统一
3. 所有专业工作台项目都已可无例外接入

## 11. 下一步动作

1. 基于本方案选定 `Studio` 的首个操作工作台验证页。
2. 将该页纳入真实 UI gate round。
3. 验证通过后，再进入 `PKC` 继承评估。
