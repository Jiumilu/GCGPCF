# GlobalCloud 绿色供应链体系界面实施排期表

日期：2026-06-08  
状态：界面实施排期表 v1  
用途：把样板页实施计划、首批组件接入实施计划、P0 最小闭环界面实施清单、高风险模块界面收口计划，统一收口为可执行排期。

---

## 1. 排期原则

界面实施排期只遵循四条规则：

1. 先 P0，后 P1/P2  
2. 先高风险链路，后一般链路  
3. 先样板页，后复制扩展  
4. 先组件接入，后视觉精修  

因此，排期不能按“哪个团队先有空”决定，而必须按：

- 业务闭环优先级
- 风险优先级
- 组件依赖关系
- 验收顺序

---

## 2. 排期阶段

### 阶段 A：样板结构冻结

目标：

1. 冻结平台订单详情页样板结构
2. 冻结签收详情页样板结构
3. 冻结统一异常处理页样板结构

完成标志：

- 三个核心样板页结构完成
- 可进入交互与组件接入

### 阶段 B：首批组件接入

目标：

1. 接入状态标签
2. 接入主/次/危险按钮
3. 接入危险确认弹窗
4. 接入证据卡片
5. 接入审计时间线

完成标志：

- 平台订单详情页样板、签收详情页样板、统一异常处理页样板完成第一轮组件接入

### 阶段 C：高风险链路交互收口

目标：

1. 收口平台订单主操作与异常入口
2. 收口签收确认与回执查看
3. 收口异常关闭与复盘入口

完成标志：

- 平台订单 -> 签收 -> 异常 三者的结构、交互、组件一致性成立

### 阶段 D：扩展到辅助样板页

目标：

1. 完成平台订单列表页样板
2. 完成证据详情页样板
3. 完成 AI 建议与治理确认样板页

完成标志：

- 证据/审计和 AI/治理交互纳入统一体系

### 阶段 E：样式统一收口

目标：

1. 统一设计令牌
2. 统一组件样式
3. 统一页面视觉层级

完成标志：

- 首批样板页进入最终交付风格基线

## 4. 不允许的排期方式

以下排期方式一律禁止：

1. 先大面积改所有页面，再慢慢统一
2. 先做最终视觉，再补结构和组件
3. 先做低风险页面，回避高风险链路
4. 组件未定义就大面积开发页面
